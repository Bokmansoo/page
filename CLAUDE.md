# Project Guidance

## Design System
Always read `DESIGN.md` before making any visual or UI decisions.

All font choices, colors, spacing, page hierarchy, and aesthetic direction are defined there. Do not deviate from `DESIGN.md` without explicit user approval.

For Sellform, creation and review flows should feel like a bright commerce production service for solo sellers, not a dark AI dashboard or generic SaaS editor.

In QA or code review mode, flag UI that conflicts with `DESIGN.md`, especially dark dashboard frames, purple/blue AI gradients, excessive AI decoration, and editor-first layouts.
