# 조사 기록: 쿠팡 및 네이버 스마트스토어 이미지·상세페이지 등록 규격

- **조사 질문:** 쿠팡 WING과 네이버 스마트스토어의 최신 상세페이지 및 썸네일(대표 이미지) 등록 규격(가로, 세로, 용량, 확장자, 해상도 등)은 무엇인가?
- **확인 날짜:** 2026-06-23
- **재확인 날짜:** 2026-12-23 (6개월 주기 갱신)
- **출처 링크:**
  - 쿠팡 판매자 센터(WING) 가이드: [Coupang Marketplace](https://marketplace.coupang.com)
  - 네이버 스마트스토어 판매자 센터 가이드: [Naver Smartstore Center](https://sell.smartstore.naver.com)
- **검증 상태:** 판매자 로그인이 필요한 세부 가이드의 원문·개정일·문단 위치를 이 Sprint에서 확보하지 못했다. 아래 수치는 구현 확정값이 아니라 기존 가설이며, 판매처 출력 기능을 구현하는 Sprint 5 전에 로그인된 공식 가이드의 직접 URL·캡처·확인 날짜로 재검증해야 한다.

---

## 1. 플랫폼별 등록 규격 (재검증 전 가설)

### 1.1 쿠팡 (Coupang) 상세설명 및 이미지 규격
- **상세설명 이미지 가로폭:** **780px 권장** (시스템 안정성 및 렌더링 호환성 기준).
- **상세설명 이미지 세로 길이:** **개별 파일당 최대 3,000px 제한**. 세로로 긴 상세페이지는 3,000px 이하로 분할하여 업로드해야 함 (최대 20장 업로드 가능).
- **이미지 개별 파일 용량:** **최대 5MB 이하**.
- **허용 파일 포맷:** `JPG`, `JPEG`, `PNG` 만 허용 (**`GIF` 애니메이션 포맷 업로드 불가능**).
- **대표 이미지 (썸네일):** **1000px × 1000px 권장** (최소 500px 이상, 1:1 정사각형 비율 강력 권장. 비율 불일치 시 자동 크롭 또는 깨짐 발생). 최대 용량 10MB 이하.

### 1.2 네이버 스마트스토어 (Naver Smartstore) 상세설명 및 이미지 규격
- **상세설명 이미지 가로폭:** **860px 권장**.
- **상세설명 이미지 세로 길이:** 공식적인 제한은 없으나 로딩 최적화를 위해 **개별 파일당 5,000px 이하** 분할 권장.
- **이미지 개별 파일 용량:** **최대 20MB 이하** (웹 성능상 500KB 이하 최적화 권장).
- **허용 파일 포맷:** `JPG`, `JPEG`, `PNG`, `GIF`, `BMP` 허용 (`JPG` 권장).
- **해상도 및 DPI:** **72 DPI 또는 96 DPI 권장**. 96 DPI를 초과하는 고해상도 이미지는 모바일/PC 색상 왜곡 및 깨짐 현상이 일어날 수 있음.
- **대표 이미지 (썸네일):** **1000px × 1000px 권장** (최소 300px × 300px, 1:1 정사각형 비율).

## 2. 구현 전 필수 재검증 항목

1. 쿠팡: 상세 설명 이미지의 허용 폭·높이·장수·용량·형식과 대표 이미지 규격
2. 스마트스토어: 상세 설명 이미지의 허용 폭·높이·용량·형식과 대표 이미지 규격
3. 두 판매처의 GIF·애니메이션·외부 링크·HTML 사용 제한
4. 원문 가이드의 개정일, 직접 URL 또는 로그인 후 경로, 확인자와 캡처

이 항목이 공식 원문으로 검증되기 전에는 어떤 숫자도 렌더링·분할·업로드 차단 규칙의 하드코딩 값으로 사용하지 않는다.

## 3. 조사 내용에 대한 해석 (해석)

1. **렌더링 해상도 대응:** 쿠팡(780px)과 네이버(860px)의 최적 가로 해상도가 상이함. 하나의 고해상도 마스터 상세페이지를 디자인한 뒤, 각 판매 채널 프리셋에 맞추어 다운스케일링 및 분할 렌더링하는 전략이 필요함.
2. **모바일 가독성 우선순위:** 두 채널 모두 모바일 트래픽이 70~80% 이상을 차지하므로, 모바일 미리보기 화면에서 텍스트가 뭉개지지 않도록 PC 폰트 크기 기준 대비 1.5배 크게 렌더링할 필요가 있음.
3. **GIF의 제한적 사용:** 스마트스토어는 상품 작동 과정이나 제형을 GIF로 업로드해 역동성을 강조할 수 있으나, 쿠팡은 정적 이미지(`JPG`, `PNG`)만 지원하므로 동일한 섹션이라도 채널별 출력 포맷 분기 처리가 필수적임.

---

## 4. 제품 구현에 미치는 영향

1. **상세페이지 조합/출력 엔진 설계 (Sprint 5 반영)**
   - Headless Chromium 렌더러는 판매처별 폭·높이·용량·형식을 설정 데이터로 받는다. 값은 위 재검증을 마친 뒤에만 프리셋으로 등록한다.
   - 단일 세로 통이미지 대신 HTML 요소 경계를 기준으로 자동 분할(Chunking)하는 구조를 택한다. 분할 높이와 장수 제한은 검증된 판매처 프리셋에서 가져온다.
2. **검수 규칙 엔진 (Sprint 5 반영)**
   - 개별 이미지 용량이 5MB(쿠팡) 및 20MB(네이버)를 초과하지 않도록 압축률을 조절하고 경고하는 검수 규칙을 추가한다.
   - GIF 파일이 포함된 상품인 경우, 쿠팡 전용 출력물을 다운로드할 때 정적 프레임으로 자동 변환하거나 업로드를 차단하는 가이드 경고를 표출한다.
