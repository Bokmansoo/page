# Sellform 홈페이지로 상세페이지 만드는 법

이 문서는 Sellform 홈페이지를 사용해서 소싱 상품 자료를 등록하고, 쿠팡/네이버 스마트스토어 등에 올릴 수 있는 상세페이지 초안을 만드는 방법을 설명한다.

## 전체 흐름

Sellform의 기본 작업 순서는 다음과 같다.

```text
서버 실행
→ 상품 프로젝트 생성
→ 상품 정보/이미지 등록
→ 상품 사실 카드 검증
→ AI 상세페이지 초안 생성
→ 섹션 직접 편집 또는 AI 재생성
→ 검수
→ 이미지 패키지 내보내기 또는 웹형 랜딩페이지 발행
```

처음에는 이 흐름만 기억하면 된다.

## 시작 전 확인

### 1. 백엔드와 프론트엔드 실행

먼저 로컬 서버가 실행되어 있어야 한다.

백엔드:

```cmd
cd C:\page
powershell -ExecutionPolicy Bypass -File .\run_backend.ps1
```

프론트엔드:

```cmd
cd C:\page\frontend
npm.cmd run dev
```

브라우저에서 접속한다.

```text
http://localhost:3000/workspace
```

서버 실행 자체가 안 되면 먼저 아래 문서를 확인한다.

- [Sellform 로컬 서버 실행 Runbook](../runbooks/2026-06-24-sellform-local-server-runbook.md)

### 2. Mock 테넌트 설정 확인

현재 Sellform은 개발용 Mock 사용자/워크스페이스 기준으로 동작한다. 왼쪽 하단에 `Mock 테넌트 설정` 카드가 보인다.

기본값은 보통 그대로 두면 된다.

- User: `00000000-0000-0000-0000-000000000001`
- Workspace: `00000000-0000-0000-0000-000000000002`

특별히 권한 테스트를 하는 상황이 아니라면 변경하지 않는다.

## 1. 상품 프로젝트 만들기

### 1-1. 새 프로젝트 화면으로 이동

브라우저에서 다음 화면으로 간다.

```text
http://localhost:3000/workspace
```

오른쪽 위의 `새 상품 프로젝트` 버튼을 클릭한다.

또는 직접 접속한다.

```text
http://localhost:3000/workspace/projects/new
```

### 1-2. 기본 정보 입력

`프로젝트 기본 정보 기입` 화면에서 다음을 입력한다.

- 프로젝트/상품 이름
- 적용 브랜드

처음에는 브랜드를 `Default Brand`로 두면 된다.

예시:

```text
프로젝트 이름: 접이식 무선 미니 선풍기
브랜드: Default Brand
```

입력 후 `다음 단계로` 버튼을 누른다.

### 1-3. 상품 자료 등록 방식 선택

상품 자료는 두 가지 방식으로 등록할 수 있다.

#### 방식 A. 공급처 URL 입력

1688, 타오바오, 알리바바, 국내 도매처 등 상품 페이지 주소를 입력한다.

```text
https://detail.1688.com/offer/...
```

그 다음 `분석 및 수집` 버튼을 누른다.

주의:

- 공급처 사이트가 로그인, 캡차, 보안 차단을 걸면 자동 수집이 실패할 수 있다.
- 자동 수집이 실패해도 프로젝트가 중단되는 것은 아니다.
- 실패하면 수동 입력 화면으로 전환된다.

#### 방식 B. 수동 자료 입력

자동 수집이 실패하거나 처음부터 직접 자료를 넣고 싶다면 수동 입력을 사용한다.

수동 입력에는 다음 자료를 넣을 수 있다.

- 공급처 상세 설명글
- 중국어/한국어 스펙표
- 제품 특징
- 옵션 정보
- 주의사항
- 상품 이미지

이미지는 다음 조건을 지킨다.

- JPG, JPEG, PNG
- 파일당 최대 10MB

입력 후 `수동 데이터로 초안 만들기` 버튼을 누른다.

## 2. 상품 사실 카드 검증하기

프로젝트가 생성되면 대시보드의 프로젝트 목록에서 해당 상품을 클릭한다.

프로젝트 상세 화면은 보통 다음 주소 형태다.

```text
http://localhost:3000/workspace/projects/{project_id}/facts
```

이 화면의 목적은 상세페이지에 사용할 수 있는 “확인된 상품 사실”을 정리하는 것이다.

### 2-1. 카테고리 확인

카테고리는 상세페이지 구조와 검수 기준에 영향을 준다.

예시:

- 패션/잡화
- 뷰티/화장품
- 식품/건강식품
- 생활/리빙

AI가 추천한 카테고리가 맞으면 확인한다. 틀리면 직접 수정한다.

### 2-2. AI 분석 실행

목표 흐름은 사용자가 모든 사실 카드를 처음부터 직접 쓰는 것이 아니라, AI가 먼저 후보를 채우고 사용자가 검수하는 방식이다.

상품 URL, 복사한 상세 설명 텍스트, 업로드 이미지가 등록되어 있다면 `AI 분석` 또는 `AI로 사실 카드 자동 생성` 버튼을 사용한다.

AI 분석은 입력된 자료에서 상세페이지에 쓸 수 있는 사실 후보를 만든다. 링크 수집이 캡차, 로그인, 보안 정책, 크롤링 제한 등으로 실패하면 상품 설명 텍스트나 이미지를 직접 넣고 다시 분석한다.

현재 화면에서 사실 카드가 자동으로 채워지지 않거나 비어 있으면 자동 추출이 아직 실패했거나 구현 보완이 필요한 상태일 수 있다. 이 경우 수동 사실 추가로 진행하되, 최종 목표는 AI 자동 후보 생성 후 사용자 검수 흐름이다.

예시:

```text
이 상품은 3단 풍속 조절을 지원합니다.
본체는 접이식 구조입니다.
USB-C 충전을 지원합니다.
```

### 2-3. 사실 카드 검증

각 사실 카드를 보고 상태를 정한다.

- 확인됨: 상세페이지에 사용해도 되는 사실
- 수정 필요: 표현이나 정보가 애매한 사실
- 미확인/제외: 근거가 부족해 사용하지 않을 사실

중요:

최종 상세페이지는 확인된 사실을 중심으로 만들어야 한다. 확인되지 않은 효능, 성능, 인증, 원산지, 수치 정보는 상세페이지에 넣지 않는 것이 안전하다.

### 2-4. 수동 사실 추가

수동 사실 추가는 기본 작업이라기보다 보완 작업이다.

AI가 놓친 정보가 있거나, 링크 자동 수집이 실패했거나, 공급처 이미지/텍스트에서 명확히 추출되지 않은 정보가 있으면 직접 사실 카드를 추가한다.

좋은 사실 카드 예시:

```text
이 상품의 색상 옵션은 화이트, 블랙, 베이지입니다.
제품 구성품은 본품 1개와 USB-C 충전 케이블 1개입니다.
상세 사이즈는 접었을 때 120mm x 80mm입니다.
```

나쁜 사실 카드 예시:

```text
무조건 잘 팔리는 상품입니다.
피로가 완전히 회복됩니다.
국내 최저가입니다.
```

## 3. 상세페이지 초안 생성하기

사실 검증이 어느 정도 끝나면 상세페이지 에디터로 이동한다.

보통 다음 주소 형태다.

```text
http://localhost:3000/workspace/projects/{project_id}/page-editor
```

처음에는 상세페이지가 아직 없으므로 `상세페이지 AI 초안 생성하기` 버튼이 보인다.

버튼을 누르면 Sellform이 다음을 기반으로 상세페이지 초안을 만든다.

- 확인된 상품 사실 카드
- 상품 카테고리
- 브랜드 설정
- 업로드된 상품 이미지
- 판매처 상세페이지 구조

## 4. 상세페이지 편집하기

상세페이지 초안이 생성되면 섹션 목록과 미리보기가 표시된다.

### 4-1. 섹션 구조 확인

상세페이지는 여러 섹션으로 구성된다.

예시:

- 헤더/메인 카피
- 상품 핵심 장점
- 사용 장면
- 상세 스펙
- 옵션/구성품
- 주의사항
- FAQ
- 배송/교환 안내

카테고리에 따라 섹션 구성은 조금씩 달라질 수 있다.

### 4-2. 직접 편집

각 섹션에서 제목과 본문 문구를 직접 수정할 수 있다.

수정할 때는 다음 기준을 지킨다.

- 확인된 사실만 사용한다.
- 과장 표현을 줄인다.
- 식품/건강식품/뷰티 제품은 효능 표현을 특히 조심한다.
- 쿠팡/스마트스토어에 올릴 수 있도록 모바일 가독성을 우선한다.

### 4-3. AI 재생성

특정 섹션의 문구가 마음에 들지 않으면 AI 재생성을 사용한다.

추천 사용법:

- 전체를 한 번에 갈아엎기보다 섹션 단위로 수정한다.
- “더 고급스럽게”, “더 짧게”, “스마트스토어 스타일로”처럼 방향을 정한다.
- 재생성 후 사실과 맞지 않는 표현이 없는지 다시 확인한다.

### 4-4. 버전 복원

상세페이지 편집 과정에서 버전 기록이 쌓인다. 이전 버전이 더 낫다면 복원 기능을 사용한다.

## 5. 상세페이지 검수하기

내보내기 전에 검수 화면으로 이동한다.

```text
http://localhost:3000/workspace/projects/{project_id}/export
```

이 화면에서는 상세페이지가 판매처 제출 가능한 상태인지 확인한다.

검수 결과는 보통 두 종류로 나뉜다.

- Blocker: 반드시 고쳐야 내보내기 가능
- Warning: 내보내기는 가능하지만 확인이 필요한 항목

예시:

```text
Blocker: 식품 카테고리에서 질병 치료 표현 사용
Warning: 특정 섹션에 이미지가 없음
```

Blocker가 있으면 먼저 상세페이지 에디터로 돌아가 수정한다.

## 6. 판매처 이미지 패키지 생성하기

검수에서 내보내기가 가능하면 `판매처 이미지 패키지 생성` 버튼을 누른다.

Sellform은 상세페이지를 이미지형 출력물로 렌더링하고 ZIP 파일로 묶는다.

현재 지원 방향:

- 쿠팡용 이미지 패키지
- 스마트스토어용 이미지 패키지

생성 완료 후 다운로드 버튼을 눌러 ZIP 파일을 내려받는다.

주의:

현재 단계에서는 쿠팡/스마트스토어에 자동 업로드하지 않는다. 다운로드한 이미지 패키지를 각 판매자센터에 직접 업로드하는 방식이다.

## 7. 웹형 랜딩페이지 발행하기

이미지형 상세페이지 외에 웹으로 볼 수 있는 공개 상세페이지도 만들 수 있다.

발행 화면:

```text
http://localhost:3000/workspace/projects/{project_id}/publish
```

이 화면에서 할 수 있는 일:

- 공개 랜딩페이지 발행
- 외부 판매처 URL 연결
- 공개 페이지 미리보기
- 발행 상태 관리

발행된 공개 페이지는 보통 다음 형태로 접근한다.

```text
http://localhost:3000/p/{slug_or_page_id}
```

공개 페이지는 이미지 갤러리, 상세 섹션, 상품 소개, 외부 구매 링크 등을 보여준다.

## 8. 운영 리포트 확인하기

왼쪽 메뉴의 `운영 리포트`로 이동한다.

```text
http://localhost:3000/workspace/operations
```

이 화면에서는 다음을 확인할 수 있다.

- 전체 프로젝트 수
- AI 작업 성공률
- Export 성공률
- 카테고리별 검수 현황
- 프로젝트별 작업 이력
- 실패한 작업 여부

처음 테스트 데이터가 필요하면 `Mock 실상품 시딩` 버튼을 사용할 수 있다.

주의:

시딩은 테스트용 상품 데이터를 만든다. 실제 운영 데이터와 섞이지 않게 개발 단계에서만 사용한다.

## 9. 브랜드/워크스페이스 설정하기

왼쪽 메뉴의 `워크스페이스 설정`으로 이동한다.

```text
http://localhost:3000/workspace/settings
```

이 화면에서는 다음을 관리한다.

- 브랜드 이름
- 브랜드 컬러
- 하단 고정 안내 문구
- 팀 멤버 초대
- 권한 역할
- 사용량/비용 제한 상태

처음에는 `Default Brand`를 그대로 써도 된다. 나중에 실제 판매 브랜드가 정해지면 브랜드 색상과 공통 안내 문구를 수정한다.

## 추천 작업 방식

처음 상품 1개를 테스트할 때는 아래 순서로 진행하는 것을 추천한다.

1. `새 상품 프로젝트` 생성
2. 상품명 입력
3. 공급처 설명글과 이미지 수동 입력
4. 사실 카드 5~10개만 먼저 정리
5. 확인된 사실만 남김
6. 상세페이지 AI 초안 생성
7. 섹션 문구를 직접 짧게 다듬음
8. 검수 화면에서 Blocker 제거
9. 이미지 패키지 생성
10. ZIP 다운로드 후 실제 판매처 업로드 가능 여부 확인

처음부터 모든 자동화를 붙이려고 하지 말고, “내가 실제로 소싱한 상품 하나를 상세페이지로 만들 수 있는지”를 먼저 확인하는 것이 좋다.

## 좋은 입력 자료 예시

수동 입력란에는 이런 식으로 넣으면 좋다.

```text
상품명: 접이식 무선 미니 선풍기
카테고리: 생활/리빙
색상: 화이트, 블랙, 베이지
충전 방식: USB-C
풍속: 3단 조절
구성품: 본품, USB-C 케이블, 설명서
특징: 접이식 구조, 탁상용 사용 가능, 휴대 가능
주의사항: 물에 닿지 않도록 주의, 충전 중 장시간 방치 금지
```

이미지는 다음을 준비하면 좋다.

- 대표 상품 이미지
- 사용 장면 이미지
- 사이즈/스펙 이미지
- 구성품 이미지
- 옵션 색상 이미지

## 주의해야 할 표현

상세페이지에는 아래 표현을 조심한다.

- “무조건 효과”
- “100% 보장”

특히 뷰티, 식품, 건강식품은 광고 심의와 플랫폼 정책을 고려해야 한다.

## 문제가 생겼을 때

### 화면에 `Failed to fetch`가 뜬다

백엔드가 꺼져 있거나 8000 포트를 다른 프로세스가 잡고 있을 가능성이 높다.

아래 문서를 보고 백엔드를 다시 실행한다.

- [Sellform 로컬 서버 실행 Runbook](../runbooks/2026-06-24-sellform-local-server-runbook.md)

### 프로젝트는 생겼는데 상세페이지 생성이 실패한다

가능한 원인:

- AI API 키가 설정되지 않음
- 확인된 사실 카드가 부족함
- 백엔드 API가 500을 반환함

우선 사실 카드를 수동으로 몇 개 추가하고 다시 시도한다.

### 이미지 패키지 생성이 실패한다

가능한 원인:

- 검수 Blocker가 남아 있음
- 렌더링 과정에서 파일 잠금이 발생함
- 상세페이지 섹션 데이터가 비어 있음

먼저 검수 화면에서 Blocker가 없는지 확인하고, 백엔드를 재시작한 뒤 다시 시도한다.

## 현재 버전에서 알아둘 점

현재 Sellform은 내부 도구/MVP 성격의 로컬 개발 버전이다.

- 실제 쿠팡/스마트스토어 자동 업로드는 아직 하지 않는다.
- 공급처 URL 자동 수집은 사이트 보안 정책에 따라 실패할 수 있다.
- 실패 시 수동 입력으로 진행하는 것이 현재 가장 안정적이다.
- AI 결과는 초안으로 보고, 최종 문구는 반드시 사람이 검수한다.
- 판매처 정책, 광고 표현, 인증/효능/원산지 정보는 실제 근거 자료를 기준으로 확인한다.
