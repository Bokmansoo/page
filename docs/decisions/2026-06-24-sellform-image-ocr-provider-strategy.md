# 결정 기록: Sellform 이미지 OCR/멀티모달 Provider 전략

- 날짜: 2026-06-24
- 상태: **승인됨 (Approved)**

## 1. 배경

Sprint 10은 이미지 자산을 사실 후보의 근거로 연결했지만, 이미지 안의 텍스트를 실제로 읽지는 않았다. 이로 인해 상세 성분표, 복합 규격표, 전성분과 같은 중요 이미지 기반 팩트들이 여전히 수동 입력에 의존하고 있어, 이미지 내 텍스트(OCR) 및 사양에 대한 사실 카드 자동 추출 기능 고도화가 요구되었다.

## 2. 후보 비교 및 대안

| 후보 | 장점 | 단점 | 1차 적용 여부 |
| --- | --- | --- | --- |
| **deterministic mock OCR** | 테스트 안정성 보장, 호출 비용 없음, 네트워크 장애로부터 독립적 | 실제 업로드 이미지의 무작위 텍스트 인지 불가능 | **적용** (1단계 기반 구조 확립용) |
| **local OCR** (Tesseract 등) | 로컬 서버 환경 구동 가능, 클라우드 비용 없음 | 다국어(한글/중국어) 및 표(Table) 인식 정확도가 떨어지며 모듈 설치 부담 큼 | **검토** (추후 확장 가능) |
| **multimodal LLM** (OpenAI 등) | 이미지 속 레이아웃, 표 구조화, 다국어 번역 성능 최고 수준 | 외부 API 호출 비용 발생, 개인정보(서류 유출 등) 및 보안 동의 정책 필요 | **후속** (2단계 서비스 도입용) |

## 3. 결정

Sprint 12 1차 구현은 **deterministic mock OCR + provider interface**로 시작한다. 실제 OCR 또는 멀티모달 LLM은 adapter 패턴으로 언제든 교체 추가 가능한 유연한 인터페이스 구조를 우선적으로 만든다.

## 4. 안전 원칙

- 이미지 분석으로 추출된 모든 사실 후보는 사용자 승인 전에는 자동으로 `confirmed` 처리하지 않는다.
- 낮은 신뢰도(Confidence) 점수를 가진 사실 카드는 기본 `needs_revision`(수정 필요) 상태로 저장한다.
- API Key나 환경 설정이 유효하지 않을 경우 자동으로 mock provider로 Fallback 한다.
- 외부 멀티모달 LLM으로 이미지 데이터를 전송하기 전에는, 민감 정보 유출 방지를 위해 사용자에게 외부 전송에 대한 명시적 동의 게이트를 확보해야 한다.
