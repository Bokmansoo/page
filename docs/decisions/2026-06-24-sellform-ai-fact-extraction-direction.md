## 1. 결정 배경

Sellform의 사실 확인 보드는 안전성을 위해 사용자가 사실을 직접 검수하도록 설계되어 있다.

하지만 실제 소싱 상품을 많이 다루면 사용자가 모든 사실 카드를 처음부터 수기로 작성하는 부담이 커진다. 상세페이지 제작 시간을 줄이려면 AI가 링크, 텍스트, 이미지에서 먼저 사실 후보를 만들고, 사용자는 검수와 확정에 집중하는 흐름이 필요하다.

## 2. 결정

Sellform은 사실 카드 단계를 다음 방향으로 고도화한다.

```text
상품 URL / 복사 텍스트 / 업로드 이미지
→ AI 사실 후보 자동 추출
→ 사용자 검수·수정·삭제·확정
→ 확인된 사실만 상세페이지 생성에 사용
```

## 3. 핵심 원칙

- AI가 만든 사실은 최종 사실이 아니라 후보이다.
- 자동 생성 후보는 기본적으로 `unknown` 또는 `needs_revision` 상태로 둔다.
- 사용자가 `confirmed`로 승인한 사실만 상세페이지 생성에 사용한다.
- URL 수집 실패는 전체 작업 실패가 아니라 수동 입력 fallback으로 처리한다.
- 캡차, 로그인, 봇 차단, 약관 우회, 비공식 자동화는 구현 범위에서 제외한다.

## 4. 기대 효과

- 사용자가 사실 카드를 수기로 작성하는 시간을 줄인다.
- 상품 URL과 공급처 설명 텍스트를 더 적극적으로 활용한다.
- 상세페이지 생성 전 검수 품질은 유지하면서 작업 속도를 높인다.
- 나중에 외부 셀러용 서비스로 확장할 때 “AI가 먼저 정리해주는 도구”라는 가치가 더 명확해진다.

## 5. 주요 위험과 대응

| 위험 | 대응 |
| --- | --- |
| AI가 없는 사실을 만들어냄 | 근거 텍스트/이미지 연결, 사용자 확인 전 사용 금지 |
| 링크 수집 실패 | 수동 텍스트/이미지 입력 fallback |
| 공급처 정책 위반 | 캡차·로그인·차단 우회 금지 |
| 효능·인증·원산지 오기재 | 위험 플래그와 `needs_revision` 기본 상태 적용 |
| 중복 사실 과다 생성 | 중복 제거 로직과 검수 UI 제공 |
