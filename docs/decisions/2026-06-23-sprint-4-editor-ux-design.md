# 결정: 가이드형 3단 편집기 레이아웃 및 렌더링 UX 설계

- **작성일:** 2026-06-23
- **작성자:** 셀폼 개발팀
- **상태:** 승인됨

---

## 1. 결정 사항
1.  **3단 화면 분할 인터페이스**:
    *   사용자가 실시간으로 편집 맥락을 잃지 않도록 **좌측(구조 제어) / 중앙(실시간 모바일 미리보기) / 우측(세부 텍스트 및 AI 프롬프트 편집)**의 3단 고정 분할 레이아웃을 채택합니다.
2.  **모바일 웹 프리뷰 스타일**:
    *   상세페이지가 주로 스마트폰 쇼핑몰 앱에서 소비되는 특성을 살려, 중앙 영역을 가로 `375px`, 세로 `720px` 크기의 가상 모바일 디바이스 프레임으로 렌더링합니다.
3.  **부분(섹션) 단위 AI 제어**:
    *   상세페이지 전체를 다시 만드는 것은 비용이 많이 들고 기존 사용자가 열심히 편집한 본문이 전부 손실되므로, **선택된 섹션만 AI 지침을 입력해 개별 재생성**하는 부분 재생성 방식을 주력 UX 기능으로 제공합니다.

---

## 2. 이유와 배경
*   **사용자 피로도 경감**: 자유 드래그 앤 드롭 캔버스는 복잡도가 너무 높고 모바일 최적화 규격을 깨뜨리기 쉽습니다. 따라서 행(Row) 기반의 순서 변경 및 숨기기 토글로 제어하는 '가이드형 편집'이 셀러의 온보딩 허들을 낮추는 데 효과적입니다.
*   **실시간 미리보기 연동**: 사용자가 우측 패널에서 문구를 한 글자 고치거나 색상을 바꾸면 즉시 중앙 모바일 뷰어에 반영되어 즉각적인 피드백 루프를 제공합니다.

---

## 3. 대안 및 트레이드오프
*   **대안: 위지윅(WYSIWYG) 방식의 인라인 캔버스 편집**
    *   **장점:** 미리보기 영역에서 직접 텍스트를 클릭해 편집할 수 있어 직관적임.
    *   **단점:** 모바일/반응형 템플릿의 CSS가 깨지기 쉽고, 모바일 노치나 프레임과의 이벤트 간섭이 심해 오작동 확률이 높음.
    *   **결정:** 안전성과 반응형 최적화를 위해 편집 입력(우측 패널)과 결과 출력(중앙 모바일 뷰)을 완전히 물리적으로 분리하는 3단 레이아웃을 확정했습니다.
