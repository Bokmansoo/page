# 결정 기록: Sellform Sprint 6 공개 페이지 발행 전략

| 항목 | 내용 |
| --- | --- |
| 일자 | 2026-06-23 |
| 상태 | 채택 |
| 범위 | 웹형 공개 랜딩페이지 발행, 공개 URL, 공개/비공개 제어 |

## 1. 결정

Sprint 6의 웹형 출력은 `PublishedPage` 레코드를 기준으로 관리한다.

- 공개 URL은 `PublishedPage.id` 또는 사용자가 지정한 `slug`로 접근한다.
- 공개 고객용 조회 API는 인증을 요구하지 않는다.
- 단, `is_active=false`인 경우 서버에서 403으로 차단한다.
- 구매 버튼은 `external_store_url`을 사용해 쿠팡/스마트스토어 등 외부 판매처로 이동한다.
- 공개 페이지의 이미지 갤러리는 해당 프로젝트에 연결된 이미지 자산 목록을 기반으로 구성한다.

## 2. 이유

- 내부 판매 도구 단계에서는 자체 쇼핑몰보다 외부 판매처 구매 링크 연결이 더 현실적이다.
- 공개/비공개 상태를 서버에서 제어해야 이미 공유된 링크도 즉시 차단할 수 있다.
- slug를 지원하면 사람이 읽기 쉬운 URL을 제공할 수 있고, slug가 없을 때는 ID 기반 URL로 안정적으로 발행할 수 있다.

## 3. 결과

- 같은 상품 프로젝트를 이미지형 출력과 웹형 공개 페이지로 함께 활용할 수 있다.
- 판매처 자동 등록 없이도 공유 가능한 모바일 랜딩페이지를 만들 수 있다.
- 추후 도메인, CDN, QR 코드, Figma/Drive/Notion MCP 연동을 붙이기 쉬운 구조가 된다.

## 4. 남은 결정

- 운영 배포 시 정적 이미지 URL을 로컬 `/uploads`로 둘지, S3/CDN 서명 URL 또는 공개 URL로 바꿀지 결정해야 한다.
- slug 정책은 현재 중복 차단 중심이며, 향후 허용 문자, 예약어, 자동 추천 규칙을 보강할 수 있다.
