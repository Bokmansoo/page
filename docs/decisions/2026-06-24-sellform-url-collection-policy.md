# Sellform URL Source Collection Policy (Sprint 16)

본 문서는 사용자가 등록한 상품 링크(URL)에서 상품의 설명 및 스펙 텍스트를 수집하는 백엔드 수집 엔진의 운영 원칙 및 실패 대응 정책을 정의하는 아키텍처 결정 레코드(ADR)입니다.

## 1. 배경 및 목적

사용자가 신규 상세페이지 제작 프로젝트를 생성할 때, 상품의 원본 링크(예: 쿠팡, 스마트스토어 등)를 제공하면 AI가 직접 링크 내 상세 내용을 읽어 사실 카드를 자동 추출하는 동선을 원합니다. 
그러나 외부 쇼핑몰 및 도매처 웹사이트들은 다양한 이유(봇 차단 정책, CAPTCHA, 동적 렌더링, 네트워크 차단)로 직접적인 크롤링이 불가능하거나 제한적인 경우가 많습니다.

본 기능은 **"시도 가능한 보조 기능"**으로서, 합법적이고 안전한 공개 웹 스크래핑을 수행하고, 실패 시에는 사용자가 수동으로 스펙 정보를 기입하거나 상세 페이지 복사본을 일괄 붙여넣도록(Sprint 15 Bulk Input) 우아한 Fallback UX를 제공하는 것을 목표로 합니다.

## 2. 수집 원칙 (Core Decisions)

1. **외부 사이트 차단 우회 정책 배제**:
   - Sellform은 로그인 세션 요구, CAPTCHA, IP 차단(403 Forbidden, 429 Too Many Requests) 등을 회피하기 위한 인위적인 우회 프록시나 CAPTCHA 해결기를 도입하지 않습니다.
   - 외부 사이트 소유자의 봇 제어 정책(robots.txt 준수 및 접근 거부)을 존중합니다.
2. **실패는 정상 경로(Graceful Fallback)로 처리**:
   - URL 수집에 실패하더라도 API 요청이 실패(500 Internal Server Error)하거나 프로젝트 분석 프로세스가 중단되지 않습니다.
   - 실패 사유(`blocked_or_forbidden`, `timeout`, `network_error`)를 진단하여 프론트엔드로 전달하고, 사용자에게 상황 설명과 함께 수동 붙여넣기 수단을 안내합니다.
3. **순수 콘텐츠 텍스트 정제 및 개인정보 배제**:
   - `html.parser.HTMLParser`를 상속한 HTMLTextExtractor를 사용하여 `script`, `style`, `nav`, `footer`, `header`, `noscript` 등의 웹사이트 구조성 태그를 완전히 제외한 본문 영역의 순수 텍스트만 추출합니다.
   - LLM 컨텍스트 한계 및 응답 속도 제어를 위해 수집된 본문 텍스트는 최대 **20,000자**로 자릅니다(Truncation).
4. **사실 확인의 주체는 사용자**:
   - 수집된 텍스트에서 AI가 자동 추출한 사실 정보는 어디까지나 '후보 카드' 상태로 제공되며, 최종 상세페이지에 반영할 것인지는 사용자가 '확인됨' 승인을 해야만 유효하게 동작합니다.

## 3. Host별 처리 방식 및 대응 전략

| Host 유형 | 분석 시도 방식 | 실패 판정 기준 | 실패 시 프론트엔드 안내 및 대응책 (Fallback UX) |
| :--- | :--- | :--- | :--- |
| **일반 공개 HTML** | httpx GET 요청 후 HTMLTextExtractor로 텍스트 추출 | 404, 500 응답 또는 10초 초과 | 주소가 정확한지 확인을 요청하고 공급처 텍스트 직접 입력을 안내합니다. |
| **쿠팡 (Coupang)** | User-Agent를 모방하여 Fetch 시도 | 403 Forbidden 응답 또는 봇 검증 페이지 감지 | "해당 링크는 접근 권한 정책에 의해 차단되었습니다. 쿠팡 상세페이지의 상품 정보를 복사하여 수동 텍스트로 붙여넣어 주세요." |
| **네이버 스마트스토어 (SmartStore)** | Fetch 시도 (대부분 API 기반 렌더링으로 본문 텍스트 비어있음) | Fetch 성공하나 데이터 부재 또는 403, 429 차단 | "스마트스토어 상세페이지는 동적 콘텐츠 보호를 적용 중입니다. 상품 상세 설명을 복사해 수동 텍스트로 붙여넣어 주세요." |
| **중국 도매처 (1688 등)** | Fetch 시도 (인코딩/네트워크 지연 발생 가능) | 10초 타임아웃 초과 또는 텍스트 미검출 | 공급처 페이지 속도를 확인하거나 스펙 정보를 복사하여 수동 텍스트/이미지 업로드로 등록해줄 것을 요청합니다. |

## 4. 기대 효과

- 크롤링 차단으로 인한 서버 리소스 낭비 및 IP 블락 리스크 최소화.
- 사용자에게 크롤링이 항상 성공할 수 없음을 명확히 인지하게 하여 시스템의 신뢰도 저하를 방지.
- 실패 시 제공되는 "상세 정보 복사본 일괄 붙여넣기" 동선을 통해 사실 정보 입력 업무 완수 가능.
