# 결정: 셀폼 1.0 주력 AI 공급자 및 서비스 모델 선정

> **결정 정정 — 2026-06-23:** 이 문서의 기존 “하이브리드 조합 최종 채택”은 공개 비교에 근거한 가설이었고, 셀폼 테스트 팩의 실측 평가가 없으므로 철회한다. 현재 유효한 결정은 **공급자 교체 가능한 `AIServiceAdapter`를 먼저 만들고, Sprint 3 전 실측 평가 결과로 모델을 선택한다**는 것이다. 기존 내용은 당시의 비교 가설로만 보존한다.

## 현재 유효한 결정

- Sprint 0에서는 특정 AI 공급자·모델을 영구 고정하지 않는다.
- Sprint 1~2는 공급자 독립적인 데이터 모델·작업 기록·어댑터 경계를 구현한다.
- Sprint 3 전에 테스트 팩 전체를 후보 모델에 실행하고, 품질·안전성·비용·지연시간 결과를 새 결정 기록에 남긴다.
- API 키는 저장소·문서·테스트 증적에 기록하지 않는다.

## 배경
셀폼은 공급처 원본 이미지 및 텍스트 자료를 해석하여 정량적 사실을 추출하고, 카테고리 규정에 부합하는 수준의 상세페이지 설명 및 마케팅 카피를 자동으로 작성해야 합니다. 또한 최종 출력 전 법적/규제적 리스크 요소를 검수하는 비동기 워크플로를 수행합니다. 
이를 위해 이미지에 대한 깊은 시각적 이해(Multimodal), 높은 신뢰도의 JSON 구조화 데이터 출력, 자연스러운 한국어 문장 구사력, 적은 지연시간(Latency)과 지속 가능한 비용 구조를 겸비한 최적의 AI 모델 선택이 필요합니다.

## 선택지

### 선택지 1: OpenAI GPT-4o 단일 통합
- **장점:** API 생태계가 가장 성숙하고 지연시간(TTFT)이 가장 짧음. 특히 `Structured Outputs` 기능을 지원하여 스키마 에러율 0% 달성 가능.
- **단점:** 마케팅 문구 생성 시 정형화되고 인위적인 AI 특유의 한글 번역투가 남는 경향이 있어 감성적 브랜딩 문장에 아쉬움이 있음.

### 선택지 2: Anthropic Claude 3.5 Sonnet 단일 통합
- **장점:** 매우 뛰어난 한국어 어휘 선택과 풍부한 카피라이팅 스타일 구현 가능. 
- **단점:** 구조화 출력을 위한 JSON Schema 준수 강제성이 상대적으로 덜함. API 토큰 단가가 가장 높으며 속도가 약간 느림.

### 선택지 3: 기능에 따른 다중 모델 하이브리드 조합 (GPT-4o + Claude 3.5 Sonnet + Gemini 1.5 Pro 폴백)
- **장점:** 각 모델의 강점을 결합. 자료 추출 및 JSON 검수에는 GPT-4o의 `Structured Outputs`를 쓰고, 브랜드 톤앤매너 문구 작성에는 Claude 3.5 Sonnet을 사용함. 대용량 이미지 묶음이나 참고용 영상 분석에는 Gemini 1.5 Pro를 연계함.
- **단점:** API 연동 코드가 복잡해지고 여러 공급자 계정 및 크레딧 관리가 필요함.

## 결정
**선택지 3(하이브리드 조합)**을 최종 채택합니다.
- **사실 추출 및 정규화 / 검수 엔진 (Sprint 2, 3, 5):** `GPT-4o` (또는 비용 절감을 위한 `GPT-4o-mini`)
- **판매용 카피라이팅 및 상세페이지 생성 (Sprint 4):** `Claude 3.5 Sonnet`
- **대량 이미지 & 영상 소스 분석 폴백:** `Gemini 1.5 Pro`

---

## 이유와 트레이드오프

### 채택 이유
1. **스키마 무결성 (GPT-4o):** 상세페이지 레이아웃 데이터는 엄밀한 JSON 구조여야 하며, 작은 포맷 오류도 웹 브라우저 렌더러 실패로 이어집니다. OpenAI의 `Structured Outputs`는 런타임 JSON 파싱 실패를 사전에 완벽히 방어합니다.
2. **카피 품질 극대화 (Claude 3.5 Sonnet):** 1인 셀러 및 대행사의 만족도를 좌우하는 요소는 생성된 문구가 즉시 판매에 쓰일 수 있을 만큼 자연스러워야 한다는 점입니다. Sonnet 3.5의 자연스러운 한국어 작문 실력은 다소의 비용 및 지연시간 격차를 정당화합니다.
3. **확장성 확보 (Gemini 1.5 Pro):** 중국 쇼핑몰 내 다수의 이미지 슬라이드나 판매 설명 영상 분석은 Gemini의 넓은 컨텍스트 창과 탁월한 네이티브 멀티모달 능력을 활용할 때 가장 효과적입니다.

### 트레이드오프 및 대응방안
- **개발 복잡도 증가:** 각기 다른 API 스펙에 종속되지 않도록 팩토리 패턴 기반의 공통 추상 어댑터(`AIServiceAdapter`)를 설계하여 코드를 깔끔히 격리합니다.
- **비용 최적화:** 토큰 요율이 저렴한 `GPT-4o-mini` 모델을 기본 추출/정제용 모델로 병용 설정할 수 있게 개발하여, 불필요한 비용 과다 지출을 차단합니다.

## 재검토 조건
1. OpenAI 또는 Google이 한국어 생성 능력이 대폭 향상된 차세대 모델을 대단히 저렴한 단가로 발표할 경우 단일 모델 통합을 재검토함.
2. Anthropic이 공식적으로 JSON Schema 강제 지원 및 100% 준수를 보장하는 API 기능을 발표할 경우, Claude 3.5 Sonnet 단일 사용으로 비용과 코드 구조를 간소화할 수 있음.
