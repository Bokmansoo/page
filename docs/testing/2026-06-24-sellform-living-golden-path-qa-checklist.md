| 항목 | 내용 |
| --- | --- |
| 작성일 | 2026-06-24 |
| 목적 | 생활/리빙 상품 1개를 링크 입력부터 상세페이지 저장까지 끝까지 통과시키며 Sellform의 실제 사용 가능성을 검증한다. |

## 1. 골든 패스 정의

이 문서에서 말하는 골든 패스는 Sellform의 가장 중요한 첫 사용 흐름이다.

```text
상품 링크 입력
→ 상품 정보 수집
→ 사실 카드 후보 생성
→ 사용자 검수
→ Living 카테고리 확정
→ 7단 설득 구조 생성
→ 스타일 후보 선택
→ 상세페이지 미리보기 생성
→ 문구 수정
→ 저장
→ export 또는 저장 완료 확인
```

이 흐름이 막힘 없이 통과되면 Sellform은 “생활/리빙 상품 하나를 실제 판매 상세페이지로 바꿀 수 있는 내부 도구”로 볼 수 있다.

---

## 2. 테스트 전 준비

### 서버 상태

- [ ] 백엔드 서버가 실행 중이다.
- [ ] 프론트엔드 서버가 실행 중이다.
- [ ] `http://localhost:3000/workspace`에 접속할 수 있다.
- [ ] 프론트에서 `Failed to fetch` 오류가 표시되지 않는다.

### 환경 설정

- [ ] `.env` 파일이 존재한다.
- [ ] `DATABASE_URL`이 설정되어 있다.
- [ ] `UPLOAD_DIR`이 설정되어 있다.
- [ ] AI API 키가 없는 경우에도 deterministic fallback으로 진행 가능한 상태다.
- [ ] AI API 키가 있는 경우 실제 AI fact extraction 경로가 동작해야 한다.

### 기준 상품

테스트 상품:

```text
루메나 휴대용 무선 냉각선풍기
```

상품 링크:

```text
https://www.coupang.com/vp/products/8717208468?itemId=25319111445&vendorItemId=92314105593
```

## 3. 단계별 QA 체크리스트

## Step 1. 상품 프로젝트 생성

### 기대 동작

사용자가 상품명과 상품 링크를 입력하면 새 상품 프로젝트가 생성된다.

### 체크 항목

- [ ] `새 상품 프로젝트` 버튼을 누를 수 있다.
- [ ] 상품명을 입력할 수 있다.
- [ ] 상품 링크를 입력할 수 있다.
- [ ] 프로젝트 생성 후 해당 프로젝트 상세 화면으로 이동한다.
- [ ] 대시보드에 생성한 프로젝트가 표시된다.
- [ ] 프로젝트 제목이 깨지지 않고 표시된다.
- [ ] 상품 링크가 저장된다.

### 실패 시 기록

| 증상 | 원인 후보 | 조치 |
| --- | --- | --- |
| 프로젝트 생성 실패 | 백엔드 미실행, API 오류, DB 오류 | 백엔드 로그 확인 |
| 생성 후 화면 이동 안 됨 | 프론트 라우팅 오류 | 브라우저 콘솔 확인 |
| 제목이 깨짐 | 인코딩 또는 저장 문제 | DB 저장값 확인 |

---

## Step 2. 상품 정보 수집 상태 확인

### 기대 동작

상품 링크나 입력 텍스트를 기반으로 Sellform이 수집 가능한 정보를 표시한다.

### 체크 항목

- [ ] `Sourcing 근거 자료` 영역이 표시된다.
- [ ] 공급처 원본 링크를 열 수 있다.
- [ ] 링크 수집이 실패한 경우 실패 이유가 사용자에게 보인다.
- [ ] 쿠팡 링크 차단/수집 실패 시에도 수동 입력 fallback 안내가 표시된다.
- [ ] 이미지가 없을 경우 “첨부된 이미지가 없습니다”처럼 명확히 안내된다.

### 합격 기준

자동 수집이 완벽하지 않아도 괜찮다. 단, 사용자가 지금 무엇을 해야 하는지 알아야 한다.

좋은 안내 예:

```text
링크 직접 수집은 아직 지원하지 않거나 실패했습니다.
입력 텍스트와 업로드 이미지를 우선 분석했습니다.
부족한 정보는 사실 카드로 수동 추가해 주세요.
```

---

## Step 3. 사실 카드 생성

### 기대 동작

AI 또는 수동 입력으로 판매 문구의 근거가 되는 사실 카드를 최소 5개 이상 확보한다.

### 필수 사실 카드 예시

아래 중 최소 5개 이상을 확보한다.

- [ ] 휴대용 무선 냉각 선풍기
- [ ] 4,800mAh 배터리
- [ ] 최대 18시간 무선 사용 가능
- [ ] FAN JET ULTRA 모델
- [ ] 충전 후 무선 사용 가능
- [ ] 외출, 사무실, 캠핑 등 이동 환경에서 사용 가능
- [ ] 생활/리빙 카테고리 상품

### 체크 항목

- [ ] `AI로 사실 카드 자동 생성` 버튼이 보인다.
- [ ] 자동 생성이 실패하면 fallback 안내가 표시된다.
- [ ] `사실 카드 수동 추가`가 가능하다.
- [ ] 여러 사실을 한 번에 입력할 수 있다.
- [ ] 중복 사실은 제외된다.
- [ ] 생성된 사실 카드는 자동 확정되지 않는다.
- [ ] 사용자가 `확인됨`, `수정 필요`, `모름` 상태를 선택할 수 있다.
- [ ] 확인된 사실 카드만 상세페이지 주요 주장에 사용된다.

### 합격 기준

- 최소 5개 이상의 사실 카드가 생성되어야 한다.
- 그중 최소 3개 이상은 `확인됨` 상태여야 한다.
- 성능/수치 관련 사실은 근거 텍스트가 함께 있어야 한다.

---

## Step 4. 카테고리 확정

### 기대 동작

AI가 카테고리를 추천하고, 사용자가 최종 카테고리를 Living으로 확정한다.

### 체크 항목

- [ ] AI 카테고리 분석 버튼이 보인다.
- [ ] 추천 카테고리가 표시된다.
- [ ] Living을 선택할 수 있다.
- [ ] `카테고리 확정` 또는 `카테고리 재확정`을 누를 수 있다.
- [ ] 확정 상태가 화면에 표시된다.

### 합격 기준

```text
카테고리: Living
확정 상태: 확정됨
```

---

## Step 5. 7단 설득 구조 생성

### 기대 동작

Living 카테고리에 맞춰 기본 7단 설득 구조가 생성된다.

### 필수 섹션

- [ ] 고객의 고민
- [ ] 핵심 해결 메시지
- [ ] 추가 장점
- [ ] 왜 이 상품이어야 할까?
- [ ] 구매 전 확인할 장점들
- [ ] 한 문장 요약
- [ ] 상품 정보

### 체크 항목

- [ ] 모든 섹션이 생성된다.
- [ ] 섹션 순서가 자연스럽다.
- [ ] 각 섹션에 제목과 본문이 있다.
- [ ] 본문이 지나치게 짧지 않다.
- [ ] `대용량 배터리입니다.`처럼 한 줄짜리 밋밋한 문구만 반복되지 않는다.
- [ ] 확인된 사실 카드가 주요 문구에 반영된다.

### 합격 기준

각 섹션 본문은 최소 1~2문장 이상이어야 한다.

좋은 예:

```text
4,800mAh 대용량 배터리로 외출 중에도 오래 사용할 수 있습니다.
최대 18시간 무선 사용이 가능해 출근길, 사무실, 캠핑처럼 이동이 많은 상황에서도 충전 부담을 줄여줍니다.
```

나쁜 예:

```text
대용량 배터리입니다.
```

## Step 6. 스타일 후보 선택

### 기대 동작

상세페이지 생성 전 스타일 후보 3개가 표시되고, 사용자가 하나를 선택한다.

### 필수 후보 구조

- [ ] 스타일 이름
- [ ] 디자인 미리보기 또는 요약
- [ ] 판매 전략 설명
- [ ] 추천 이유
- [ ] 채널 적합도
- [ ] AI 추천 배지

### 기대 후보 예시

| 후보 | 설명 |
| --- | --- |
| 문제 해결형 | 고객의 더위/휴대성/배터리 고민을 먼저 제기하고 해결 메시지로 설득 |
| 스펙 강조형 | 4,800mAh, 최대 18시간 등 비교 가능한 수치를 전면 배치 |
| 라이프스타일형 | 출근길, 사무실, 캠핑 등 사용 장면 중심으로 설득 |

### 체크 항목

- [ ] 후보가 3개 표시된다.
- [ ] 하나의 후보에는 `AI 추천` 배지가 있다.
- [ ] 쿠팡/스마트스토어 적합도 표시가 있다.
- [ ] 선택하지 않으면 상세페이지 생성이 진행되지 않는다.
- [ ] `다른 스타일 다시 추천`이 가능하다.

---

## Step 7. 상세페이지 미리보기 생성

### 기대 동작

선택한 스타일을 기준으로 모바일 세로형 상세페이지 미리보기가 생성된다.

### 체크 항목

- [ ] page-editor로 이동할 수 있다.
- [ ] 모바일 세로형 미리보기가 표시된다.
- [ ] 7단 섹션이 모두 보인다.
- [ ] 제목/본문이 화면에서 잘리지 않는다.
- [ ] 섹션 순서를 변경할 수 있다.
- [ ] 섹션별 문구를 수정할 수 있다.
- [ ] 새 섹션을 추가할 수 있다.
- [ ] 불필요한 섹션을 숨기거나 삭제할 수 있다.

### 합격 기준

사용자가 “이 상태에서 조금만 다듬으면 올릴 수 있겠다”고 느껴야 한다.

---

## Step 8. 위험 문구 검수

### 기대 동작

근거 없는 위험 문구가 있으면 요약과 상세 패널에서 확인할 수 있다.

### 체크 항목

- [ ] 상단에 검수 요약이 표시된다.
- [ ] `주의 필요 N건`이 표시된다.
- [ ] 섹션별 근거 카드가 표시된다.
- [ ] 위험 문구가 있으면 사유가 표시된다.
- [ ] 수정 제안이 표시된다.

### 위험 문구 예시

| 문구 | 위험 사유 |
| --- | --- |
| 업계 최고 냉각 성능 | 비교 근거 없음 |
| 1초 만에 체감 온도 -10도 | 수치 근거 없음 |
| 어린이도 안전하게 사용 가능 | 안전 인증 근거 없음 |
| 100% 만족 | 보장 근거 없음 |

### 합격 기준

성능, 수치, 효능, 안전, 건강, 인증, 비교 우위 표현은 근거 없으면 경고되어야 한다.

---

## Step 9. 버전 저장

### 기대 동작

AI 생성 또는 수정된 상세페이지가 버전으로 저장된다.

### 체크 항목

- [ ] AI 초안 생성 후 자동 저장 버전이 생긴다.
- [ ] 버전명이 표시된다.
- [ ] 생성 시간이 표시된다.
- [ ] 이전 버전을 복원할 수 있다.
- [ ] 현재 버전을 최종본으로 지정할 수 있다.

### 합격 기준

최소 1개 이상의 상세페이지 버전이 저장되어야 한다.

---

## Step 10. Export 또는 저장 완료 확인

### 기대 동작

최종본을 긴 세로 이미지 또는 섹션별 이미지 ZIP으로 저장할 수 있다. Export가 아직 미완성인 경우 저장 완료 상태가 명확해야 한다.

### 체크 항목

- [ ] 최종본이 지정되어 있다.
- [ ] 긴 세로 이미지 export 버튼이 보인다.
- [ ] 섹션별 이미지 ZIP export 버튼이 보인다.
- [ ] export 전 체크리스트가 표시된다.
- [ ] export 실패 시 이유가 표시된다.
- [ ] export가 아직 미완성이라면 “저장 완료” 상태라도 명확히 확인된다.

### 합격 기준

초기에는 export가 완벽하지 않아도 된다. 단, 사용자는 생성한 상세페이지를 잃어버리지 않아야 한다.

## 4. 최종 합격 기준

Living 골든 패스는 아래 조건을 만족하면 통과로 본다.

- [ ] 상품 프로젝트가 생성된다.
- [ ] 상품 링크가 저장된다.
- [ ] 사실 카드가 최소 5개 이상 생성된다.
- [ ] 최소 3개 이상의 사실 카드가 `확인됨` 상태다.
- [ ] 카테고리가 Living으로 확정된다.
- [ ] 7단 설득 구조가 생성된다.
- [ ] 스타일 후보 3개 중 하나를 선택할 수 있다.
- [ ] 상세페이지 미리보기가 정상 표시된다.
- [ ] 문구가 지나치게 짧거나 빈약하지 않다.
- [ ] 위험 문구 검수 결과를 확인할 수 있다.
- [ ] 상세페이지 버전이 저장된다.
- [ ] 최종본을 지정할 수 있다.
- [ ] export 또는 저장 완료 상태를 확인할 수 있다.

## 5. 테스트 중 발견 이슈 기록 양식

| 번호 | 단계 | 증상 | 기대 동작 | 실제 동작 | 심각도 | 조치 방향 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | 예: 사실 카드 생성 | 예: AI 생성 실패 | 예: fallback 안내 표시 | 예: alert만 표시 | Major | UX 문구 개선 |

심각도 기준:

- Blocker: 골든 패스 진행 불가
- Major: 진행은 가능하지만 사용자가 크게 헷갈림
- Minor: 품질 개선 필요
- Nit: 문구/디자인 세부 수정

## 6. Sprint 22 후보 범위

이 체크리스트를 실제로 돌린 뒤 발견된 문제 중 가장 치명적인 3개를 Sprint 22로 묶는다.

예상 Sprint 22 후보:

- 링크 입력 후 상태 안내 개선
- 사실 카드 자동/일괄 생성 UX 안정화
- Living 7단 구조 문구 품질 개선
- page-editor 다음 행동 안내 개선
- 저장 완료/최종본 상태 표시 개선
- 생활/리빙 샘플 상품 1개 끝까지 통과
