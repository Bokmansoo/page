# 셀폼(Sellform) 최종 제품 기획서

> 상태: 초안 — 사용자 검토 필요  
> 기준일: 2026-06-23  
> 제품 한 줄: **상품 정보를, 팔리는 페이지로.**

## 1. 제품 정의

셀폼은 공급처 자료와 직접 촬영 자료를 판매 가능한 상품 정보로 정리하고, 카테고리에 맞는 상세페이지를 생성·편집·검수하여 판매 채널용 결과물로 내보내는 AI 상품 콘텐츠 스튜디오다.

첫 사용자는 상품을 직접 소싱·판매하는 운영자다. 내부 도구로 실제 상품을 반복 제작하며 검증한 뒤, 외부 셀러와 대행사까지 지원하는 구독형 서비스로 확장한다.

### 1.1 완결된 1.0의 성공 기준

사용자는 하나의 상품에 대해 다음을 끝까지 수행할 수 있어야 한다.

1. 공급처 링크·사진·텍스트 또는 직접 촬영 사진을 등록한다.
2. AI가 원본 자료를 번역·구조화하고 카테고리를 추천한다.
3. 사용자가 판매 사실을 확인·수정한다.
4. AI가 카테고리에 맞는 상세페이지 구조와 스타일을 제안한다.
5. 사용자가 AI 수정과 제한적 직접 편집으로 결과를 다듬는다.
6. 이미지형 상세페이지와 선택형 인터랙티브 웹페이지를 출력한다.
7. 이미지형 결과물을 쿠팡·네이버 스마트스토어 등록 작업에 사용할 수 있다.

### 1.2 제품 원칙

- **사실 우선:** AI는 확인되지 않은 사실, 효능, 수치, 인증을 판매 문구로 단정하지 않는다.
- **카테고리 인식:** 공통 엔진 위에 카테고리별 질문, 섹션, 검수 규칙을 적용한다.
- **AI 제안, 사용자 결정:** AI가 초안·추천·경고를 제공하고 사용자가 최종 확정한다.
- **한 번의 입력, 여러 출력:** 한 상품 프로젝트의 데이터로 이미지형·웹형 결과물을 모두 만든다.
- **완결된 내부 도구:** 1.0은 기능 데모가 아니라 실제 판매 업무를 끝낼 수 있는 도구여야 한다.
- **확장 가능한 경계:** 외부 셀러·팀·자동 등록 기능을 나중에 추가해도 데이터와 워크플로를 다시 만들지 않는다.

## 2. 대상 사용자와 핵심 상황

### 2.1 1차 사용자: 내부 셀러

- 중국·해외·국내 도매처에서 소싱한 상품을 판매한다.
- 공급처 이미지와 설명을 갖고 있지만, 번역·정리·차별화된 상세페이지 제작에 시간이 든다.
- 샘플이 있으면 직접 촬영 사진을 추가하고, 없으면 공급처 자료만으로도 초안을 만들고 싶다.
- 쿠팡과 네이버 스마트스토어에 올릴 수 있는 결과물이 필요하다.

### 2.2 향후 사용자: 외부 셀러·대행사

- 여러 브랜드와 상품을 반복 제작한다.
- 브랜드 톤, 자주 쓰는 고지 문구, 템플릿을 유지해야 한다.
- 협업, 검토, 대량 생성, 권한 관리가 필요하다.

## 3. 최종 사용자 경험

```text
새 상품 프로젝트
  → 자료 가져오기
  → AI 정리·번역
  → 사실 확인
  → 카테고리·스타일 추천
  → 상세페이지 생성
  → 편집·검수
  → 이미지형 / 웹형 출력
```

### 3.1 새 상품 프로젝트

사용자는 상품명(알고 있다면), 공급처 링크, 사진, 캡처, 텍스트 중 가능한 자료부터 제공한다. 링크 추출에 실패해도 사진과 텍스트만으로 다음 단계로 진행할 수 있어야 한다.

### 3.2 자료 정리와 사실 확인

AI는 원본 자료에서 상품명, 옵션, 규격, 재질·성분, 구성품, 원산지, 사용법, 보관·배송 정보를 추출해 한국어 구조화 데이터로 제안한다. 사용자는 각 사실을 `확인됨`, `수정 필요`, `모름` 상태로 표시한다.

`확인됨`만 판매 사실로 확정된다. `모름`은 상세페이지에 단정적으로 사용하지 않으며, 필요하면 사용자에게 추가 확인을 요청한다.

### 3.3 스타일 선택과 생성

AI는 사진, 카테고리, 가격대, 타깃, 브랜드 톤을 근거로 다음 두 가지를 제안한다.

- 추천 스타일 1개: 추천 이유를 함께 제공한다.
- 대안 스타일 1개: 다른 설득 방향을 비교할 수 있게 제공한다.

사용자는 선택 후 상세페이지를 생성한다. 스타일은 고정된 단일 디자인이 아니라 레이아웃·색상·타이포그래피·카피 톤의 조합이다.

### 3.4 편집과 검수

초기 편집기는 다음을 지원한다.

- 문구 클릭 편집, 사진 교체, 섹션 숨김·순서 변경
- 대표색·폰트 톤 변경
- 자연어 수정: 예) “더 프리미엄하게”, “20대 여성 선물용으로”, “핵심만 짧게”
- 카테고리별 누락 항목, 위험한 표현, 이미지 품질 경고
- 생성 버전 비교와 이전 버전 복원

자유 배치형 캔버스 편집은 초기 범위에서 제외한다. 템플릿 기반의 안정적인 출력 품질을 먼저 보장한다.

### 3.5 출력 방식

#### 이미지형 판매처 출력

- 긴 세로형 상세페이지를 섹션 단위로 렌더링하고, 판매처 등록에 적합한 이미지 묶음으로 내보낸다.
- 출력 전에 모바일 스크롤 미리보기와 잘림·저해상도·누락 검사를 제공한다.
- 쿠팡·네이버 스마트스토어는 초기에는 직접 등록한다. 자동 등록은 후속 고도화다.

#### 인터랙티브 웹형 출력

- 동일한 상품 프로젝트로 모바일 랜딩페이지를 발행한다.
- 기본 인터랙션은 구매 버튼 연결, 이미지 갤러리, FAQ 펼치기다.
- 옵션 전환, 전후 비교, 영상은 상품·카테고리에 따라 선택적으로 활성화한다.
- 구매 버튼은 쿠팡 또는 스마트스토어 상품 페이지로 이동한다.

## 4. 카테고리 엔진

셀폼은 공통 섹션 엔진과 카테고리별 프리셋을 결합한다. 하나의 상품은 하나 이상의 후보 카테고리를 가질 수 있지만, 상세페이지 생성 전 사용자가 최종 카테고리를 확정한다.

| 카테고리 | 중점 입력 | 기본 설득 섹션 | 주요 검수 포인트 |
|---|---|---|---|
| 패션·잡화 | 소재, 색상, 치수, 사이즈, 구성품 | 착용/활용, 소재, 사이즈표, 스타일 제안 | 사이즈·색상 오인 가능성 |
| 뷰티·화장품 | 전성분, 사용법, 용량, 피부 관련 고지 | 텍스처, 사용 순서, 성분, 주의사항 | 의학적·과장 효능 표현 |
| 식품·건강식품 | 원산지, 원재료, 보관법, 섭취 정보 | 원료, 맛/섭취, 보관, 영양 정보 | 건강 효능·표시 누락 |
| 생활·리빙 | 규격, 재질, 설치·관리 방법 | 문제 해결, 사용 장면, 비교, 규격 | 안전·호환성·치수 단정 |

카테고리 프리셋은 다음을 정의한다.

- 필요한 사실 필드와 사용자 질문
- 권장/선택/비권장 섹션
- 카피 톤과 스타일 후보
- 표현 금지·주의 규칙
- 출력 전 체크리스트

## 5. 사진과 출처 정책

사진은 `공급처 제공`, `직접 촬영`, `AI 보정`으로 분류한다. 각 사진에는 원본 출처, 사용 허용 여부 메모, 옵션 연결 정보, 검증 상태를 저장한다.

- **공급처 제공 사진:** 제품 외형·옵션·기본 구조를 보여주는 용도다.
- **직접 촬영 사진:** 실제 크기감, 사용 장면, 질감, 구성품 등 신뢰를 높이는 용도다.
- **AI 보정 이미지:** 배경 정리, 크롭, 밝기·해상도 보정까지를 우선 지원한다. 실제 제품의 형태·기능을 바꾸는 생성 편집은 별도 고도화로 둔다.

샘플이 없는 경우에는 직접 검증을 전제하는 섹션을 기본적으로 제외하고, 확인된 공급처 사실 중심으로 결과를 생성한다.

## 6. 화면 구조

1. **대시보드** — 브랜드별 상품 프로젝트 목록, 상태, 최근 출력물
2. **새 상품 마법사** — 자료 입력, 추출 진행, 카테고리 추천
3. **사실 확인 보드** — 원본 근거와 정리된 한국어 사실을 나란히 비교·확정
4. **스타일 추천 화면** — AI 추천안과 대안 비교, 추천 근거 표시
5. **상세페이지 편집기** — 섹션 목록, 캔버스 미리보기, 속성 편집, AI 수정 패널
6. **검수 센터** — 누락 정보, 위험 표현, 사진 품질, 출력 적합성 경고
7. **출력 센터** — 이미지형·웹형 선택, 판매처 링크 설정, 다운로드·공유
8. **브랜드 설정** — 로고, 색상, 폰트 톤, 기본 문구, 기본 판매처 링크

## 7. AI 워크플로와 에이전트 경계

셀폼은 자율적인 멀티에이전트 대화보다, 명시적인 상태 전이와 사람 승인 지점을 가진 파이프라인으로 설계한다.

| 단계 | 담당 | 입력 | 출력 | 사람 승인 |
|---|---|---|---|---|
| 자료 추출 | 추출·번역 AI | 링크, 이미지, 텍스트 | 원본 인용이 있는 후보 사실 | 아니오 |
| 사실 정규화 | 상품 정보 AI | 후보 사실, 카테고리 | 구조화된 상품 카드 | 예 |
| 카테고리·스타일 기획 | 기획 AI | 확정 사실, 사진, 브랜드 | 카테고리, 섹션 계획, 스타일 2안 | 예 |
| 카피 생성 | 카피 AI | 확정 사실, 섹션 계획 | 섹션별 한국어 카피 | 편집 시 |
| 레이아웃 조합 | 일반 코드 | 템플릿, 카피, 자산 | 렌더 가능한 페이지 문서 | 아니오 |
| 품질 검수 | 규칙 + 검수 AI | 페이지, 사실, 렌더 이미지 | 경고와 수정 제안 | 최종 |
| 출력 | 일반 코드 | 확정 페이지 | 이미지 묶음, 웹형 페이지 | 아니오 |

### 7.1 AI 안전장치

- 모든 생성 카피는 확인된 사실 ID를 근거로 연결한다.
- 근거가 없는 숫자·효능·인증·비교 우위 표현은 생성 금지 또는 검수 경고 대상이다.
- 원본 공급처 표현과 셀폼이 생성한 판매 문구를 구분해 보관한다.
- 사용자의 최종 확정과 편집 이력을 버전으로 기록한다.

### 7.2 LangChain·LangGraph 활용 원칙

- 초기에는 Python 서비스 내부의 명시적 워크플로와 구조화된 데이터 모델로 시작한다.
- LangChain은 모델 호출, 프롬프트, 도구 연결을 표준화할 필요가 생길 때 선택적으로 사용한다.
- LangGraph는 사용자 승인 대기, 재시도, 수동 입력 분기, 장기 실행 작업, 검수 반려 루프가 복잡해질 때 도입한다.
- 핵심 비즈니스 데이터와 상태 전이는 특정 AI 프레임워크에 종속하지 않고 데이터베이스에 저장한다.

## 8. 데이터 모델

최초부터 다중 사용자·다중 브랜드를 수용하는 경계를 둔다. 내부 도구 단계에서는 사용자 1명·브랜드 1개로 시작해도 된다.

```text
Workspace
  └─ Brand
      └─ ProductProject
          ├─ SourceDocument (링크, 텍스트, 캡처)
          ├─ Asset (공급처/직접 촬영/AI 보정 사진)
          ├─ ProductFact (값, 근거, 검증 상태)
          ├─ CategoryAssignment
          ├─ PagePlan (섹션 계획, 스타일 선택)
          ├─ PageVersion (카피, 레이아웃, 자산 연결)
          ├─ ReviewIssue
          └─ Export (이미지형, 웹형)
```

핵심 엔터티는 다음 필드를 최소로 가져야 한다.

- `Workspace`: 소유자, 향후 구독·권한 경계
- `Brand`: 브랜드명, 로고, 색상, 폰트 톤, 기본 고지 문구
- `ProductProject`: 상품 상태, 카테고리, 판매처 링크, 현재 버전
- `ProductFact`: 키, 값, 단위, 근거 원본, 검증 상태, 확정자
- `Asset`: 저장 위치, 출처, 옵션 연결, 사용 권한 메모, 품질 상태
- `PageVersion`: 템플릿 버전, 섹션 데이터, 생성 근거, 변경 이력
- `ReviewIssue`: 심각도, 규칙, 대상 섹션, 해결 상태
- `Export`: 형식, 규격 프리셋, 파일, 생성 시각, 게시 URL

## 9. 기술 아키텍처

### 9.1 권장 스택

| 영역 | 권장 기술 | 이유 |
|---|---|---|
| 프론트엔드 | Next.js, TypeScript, React | 단계형 앱, 편집 UI, 서버 렌더링과 운영 편의성 |
| UI 시스템 | Tailwind CSS + 컴포넌트 라이브러리 | 빠르고 일관된 제품 UI |
| 백엔드 API | Python + FastAPI | AI·이미지 처리 생태계와 타입 기반 API 구현 |
| 관계형 데이터 | PostgreSQL | 프로젝트·버전·근거·권한 관계를 안정적으로 관리 |
| 파일 저장 | S3 호환 오브젝트 스토리지 | 원본 사진, 렌더 결과, 공개 웹 자산 저장 |
| 비동기 작업 | Redis + 작업 큐 | 추출, AI 생성, 렌더링 같은 장기 작업 처리 |
| 렌더링 | HTML/CSS 템플릿 + Headless Chromium | 웹형 페이지와 이미지형 상세페이지를 같은 소스에서 출력 |
| AI 추상화 | 공급자 교체 가능한 어댑터 + JSON 스키마 | 특정 모델·공급자 종속 없이 구조화된 결과 보장 |
| 관측성 | 구조화 로그, 작업 추적, 오류 알림 | AI 작업 실패·비용·출력 품질 추적 |

### 9.2 서비스 경계

```text
Web App (Next.js)
  ↕ 인증된 API
Core API (FastAPI)
  ├─ PostgreSQL: 프로젝트·사실·버전·권한
  ├─ Object Storage: 이미지·출력물
  ├─ Job Queue: AI·렌더링 작업
  ├─ AI Adapter: 비전·번역·카피·검수
  └─ Renderer: HTML → 이미지 묶음 / 웹형 페이지
```

### 9.3 구현 원칙

- 모든 AI 응답은 자유 텍스트가 아니라 검증 가능한 JSON 스키마로 받는다.
- 오래 걸리는 작업은 요청-응답 경로에서 분리하고 진행 상태를 사용자에게 표시한다.
- 페이지는 섹션 JSON에서 생성한다. 최종 이미지는 원본 데이터가 아니라 렌더 산출물이다.
- 판매처별 규격은 코드에 흩어놓지 않고 출력 프리셋 설정으로 관리한다.
- AI 공급자·모델 세부 선택은 구현 직전에 최신 공식 문서와 비용·품질 평가를 거쳐 확정한다.

## 10. 보안, 권한, 신뢰

- 워크스페이스·브랜드 단위의 데이터 격리를 처음부터 적용한다.
- 원본 사진과 출력 파일은 권한 있는 사용자만 접근 가능한 서명 URL로 제공한다.
- 공급처 원본과 사진 출처·사용 권한 메모를 보관한다.
- AI 생성 결과, 사용자 수정, 최종 확정자·시각을 감사 이력으로 남긴다.
- 외부 셀러 공개 전에는 개인정보 처리, 파일 보관 정책, 약관, AI 생성물 고지 방식을 별도 검토한다.

## 13. 1.0 검증 지표

- 실제 소싱 상품 10~20개로 처음부터 끝까지 프로젝트를 완성한다.
- 각 상품은 판매처에 올릴 수 있는 이미지형 결과물을 최소 1개 생성한다.
- 생성 후 사용자가 수정한 사실·카피·섹션의 이유를 기록해 다음 개선 우선순위를 정한다.
- 자료 입력부터 첫 출력까지 걸린 시간과, 기존 수작업 대비 절감 시간을 측정한다.
- 확인되지 않은 사실을 최종 상세페이지에 단정적으로 쓴 사례를 검수 이슈로 기록·분석한다.

## 15. CEO 리뷰로 확정된 운영·품질 기준

### 15.1 사실 사용 정책

- 미확인 사실은 경고 표시가 있는 초안에서만 임시로 볼 수 있다.
- 판매처용 최종 출력에는 `확인됨` 상태의 사실만 자동으로 사용한다.
- 사용자가 미확인 사실을 포함하려면 명시적으로 선택하고 사유·이력을 남긴다.

### 15.2 실패 복구와 보안

- `SOURCE_EXTRACTION_UNAVAILABLE`(로그인 필요, 접근 제한, 빈 페이지, 지원하지 않는 형식, 시간 초과 등) 발생 시 사진 업로드·텍스트 붙여넣기·옵션표 입력으로 즉시 전환한다.
- 외부 링크는 공개 HTTP/HTTPS만 허용하며 내부 주소, 비정상 리디렉션, 과대 파일, 허용하지 않은 파일 형식을 차단한다.
- 링크·파일 처리 실패는 사용자에게 다음 행동을 안내하고, 이미 입력한 프로젝트 자료는 보존한다.

### 15.3 상태·버전·API 원칙

- 프로젝트는 생성 즉시 초안으로 저장하고, 입력·사실 확인·편집 변경은 자동 저장한다.
- AI·렌더링 작업은 중복 방지 키와 작업 ID를 사용한다. 새로고침과 재시도는 기존 작업을 재개한다.
- 사실, 상세페이지 버전, 출력물은 당시의 근거·템플릿·카피를 고정 참조해 재현 가능해야 한다.
- API는 프로젝트·자료·사실·페이지 버전·검수 이슈·출력물을 독립 리소스로 관리하며 장기 작업은 비동기로 처리한다.

### 15.4 코드·테스트·운영 기준

- 자료 수집, 사실 관리, 카테고리 규칙, AI 생성, 페이지 조합, 검수, 출력은 독립 모듈로 유지한다.
- 패션·뷰티·식품·리빙의 정상·누락·위험 입력을 포함한 상품 테스트 팩과 출력 스크린샷 비교 테스트를 유지한다.
- 모든 AI·렌더링 작업은 작업 ID, 결과 버전, 오류 원인, 소요 시간, 비용을 추적한다.
- AI 호출·렌더링 실패·파일 저장 실패는 운영 알림 대상이다.

### 15.5 성능·UX 기준

- AI 수정은 전체 페이지 재생성보다 섹션 단위 부분 재생성을 우선한다.
- 미리보기·AI 분석에는 최적화된 이미지 사본을 쓰고, 원본 파일은 별도 보관한다.
- 프로젝트별 파일 크기·사진 수·생성 횟수와 워크스페이스별 사용량 한도를 관리한다.
- 편집기는 섹션 목록, 모바일 미리보기, 선택 섹션 편집·AI 수정, 행동 가능한 검수 이슈를 갖춘 가이드형 3단 구조를 기본으로 한다.
