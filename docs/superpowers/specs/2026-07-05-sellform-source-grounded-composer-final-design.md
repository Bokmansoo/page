# Sellform 자료 기반 조립형 AI 상세페이지 최종 기획

> 작성일: 2026-07-05  
> 상태: 최종 방향 확정안  
> 핵심 문장: 사용자는 자유롭게 입력하고, AI는 자료를 구조화하고, 사용자는 생성 전에 확인하며, Sellform은 확인된 자료와 텍스트 레이어로 상세페이지를 조립한다.

## 1. 최종 제품 방향

Sellform은 "AI가 상세페이지 이미지를 통째로 그리는 도구"가 아니라, **상품 자료를 바탕으로 판매 가능한 상세페이지 문서를 조립하는 도구**로 간다.

사용자는 최소한의 자료만 있어도 시작할 수 있어야 한다. 사진 1장만 올려도 초안을 만들 수 있고, 상품 URL이나 설명, 추가 사진, 스펙, 참고 상세페이지를 넣으면 결과가 더 정확하고 풍부해진다.

최종 결과물은 먼저 구조화된 상세페이지 문서다. 긴 PNG/JPG는 그 문서를 브라우저에서 렌더링한 뒤 캡처해서 만든다. 이 원칙을 지키면 한글이 깨지지 않고, 생성 후 수정도 가능하며, 이미지 생성 비용도 필요한 곳으로 제한할 수 있다.

## 2. 왜 이 방식인가

상세페이지에서 이미지 생성 모델에게 한글 문구까지 그리게 하면 글자가 깨지거나 실제 상품 정보가 왜곡될 가능성이 높다. 특히 상품명, 성분, 인증, 가격, 스펙, 주의사항처럼 정확해야 하는 정보는 이미지 안에 직접 생성하면 안 된다.

반대로 단순 템플릿만 쓰면 결과가 밋밋하고 "색상 배경 위에 상품 이미지만 얹은" 느낌이 난다. 사용자가 원하는 결과는 참고 상세페이지처럼 섹션마다 목적이 다르고, 상품 컷, 사용 장면, 비교, 성분/스펙, 인증, 추천 대상, CTA가 리듬 있게 이어지는 상업용 상세페이지다.

그래서 최종 방식은 하이브리드다.

```text
LLM = 상품 이해, 사실 추출, 카피, 스토리보드 생성
Extractor/OCR = URL과 이미지에서 자료 수집
Image generation = 텍스트 없는 장면/배경/보조 컷 생성
Compositor = 원본 상품 누끼와 AI 배경 합성
React renderer = 한글 텍스트, 카드, 표, 아이콘, 이미지 조립
Playwright = 최종 PNG/JPG export
```

## 3. 최종 입력 UX

첫 화면은 단순해야 한다. 사용자가 처음부터 많은 칸을 채우게 만들면 이탈 가능성이 높다.

### 3.1 기본 입력

첫 화면에는 아래 세 가지만 크게 보여준다.

```text
상품 자료를 넣어주세요

[상품 사진 업로드]
[상품 URL 또는 상품 설명을 자유롭게 붙여넣는 큰 입력창]
[상세페이지 초안 만들기]
```

사진 1장, URL 1개, 설명 텍스트 중 하나만 있어도 시작할 수 있다. 다만 이미지가 하나도 없고 텍스트만 있는 경우에는 이미지 정확도가 낮다는 안내를 보여준다.

### 3.2 "처음엔 한 박스, 생성 전에는 칸칸이 확인"

"처음엔 한 박스"는 사용자가 상품명, 스펙, 가격, 배송, 원하는 분위기, 참고 URL을 구분해서 입력하지 않아도 된다는 뜻이다. 자유 입력창 하나에 아는 내용을 그대로 붙여넣게 한다.

예:

```text
아이용 자전거입니다. LED 조명이 있고 보조바퀴 탈착 가능해요.
가격은 39,900원이고 무료배송입니다.
안전하고 감성적인 느낌의 상세페이지로 만들어주세요.
참고 URL은 https://example.com/product/123 입니다.
```

그러면 AI가 이 내용을 아래처럼 구조화한다.

```text
상품명 후보: 아이용 LED 자전거
가격: 39,900원
배송: 무료배송
핵심 특징:
- LED 조명
- 보조바퀴 탈착 가능
- 아이용 안전 자전거
원하는 분위기: 안전함, 감성적
참고 URL: https://example.com/product/123
```

"생성 전에는 칸칸이 확인"은 AI가 구조화한 정보를 상세페이지 생성 전에 사용자에게 필드별로 보여주고, 사용자가 맞는 정보만 확인하거나 수정하게 한다는 뜻이다.

### 3.3 더 정확하게 만들기

기본 입력 아래에는 접힌 고급 영역을 둔다.

```text
+ 더 정확하게 만들기
```

펼치면 다음 필드를 제공한다.

```text
상품명
상품 설명
스펙/성분
가격/배송 정보
추가 사진
상품 URL
참고 상세페이지 URL
원하는 분위기
판매 채널
강조 문구
금지 문구
```

이 필드는 필수가 아니다. 입력하면 결과 품질과 정확도가 올라간다는 안내만 한다.

## 4. 입력별 품질 정책

Sellform은 입력 자료의 양과 신뢰도에 따라 결과 범위를 다르게 잡는다.

### 4.1 사진 1장만 있는 경우

가능한 결과:

```text
- 원본 상품 중심 히어로
- 원본 누끼를 활용한 장점 섹션
- 텍스트 기반 문제/해결 카드
- 추천 대상
- FAQ
- 제품 정보 표
- 일부 AI 배경/장면 보조 컷
```

주의:

```text
- 정확한 측면/후면 컷은 보장하지 않는다.
- 패키지, 구성품, 인증서, 성분표는 원본 자료가 없으면 생성하지 않는다.
- AI가 상상한 사실을 상품 스펙처럼 쓰지 않는다.
```

### 4.2 여러 사진이 있는 경우

가능한 결과:

```text
- 정면/측면/후면/구성품을 구분해서 섹션에 배치
- 상세 컷, 비교 컷, 구성품 컷의 정확도 상승
- 원본 사진 크롭과 누끼 합성으로 반복 느낌 감소
```

### 4.3 상품 URL이 있는 경우

가능한 결과:

```text
- HTML에서 상품명, 가격, 스펙, 설명 추출
- 상세 이미지에서 OCR로 문구 후보 추출
- URL 이미지 후보 수집
- 출처가 있는 사실 카드 생성
```

제약:

```text
- 로그인, 캡차, 크롤링 방어, 동적 로딩 때문에 일부 URL은 실패할 수 있다.
- 타사 상세페이지 문구를 그대로 복사하지 않는다.
- 참고 상세페이지는 구조와 흐름을 참고하되, 문구와 디자인을 그대로 복제하지 않는다.
```

### 4.4 상품 설명/스펙이 있는 경우

가능한 결과:

```text
- 카피 정확도 상승
- 성분/스펙/주의사항/배송 정보 섹션 생성 가능
- 스마트스토어/쿠팡 등록 정보와 연결 가능
```

## 5. 상세페이지 생성 원칙

### 5.1 이미지 모델은 글자를 만들지 않는다

이미지 생성 프롬프트에는 항상 아래 정책을 포함한다.

```text
No text, no Korean letters, no English letters, no logo, no watermark, no label.
Leave clean space for typography.
```

한글 제목, 설명, 성분명, 스펙표, 체크리스트, CTA는 모두 React/HTML 텍스트 레이어로 렌더링한다.

### 5.2 상품 정확도는 원본 자료가 우선이다

상품의 외형, 라벨, 구성품, 인증, 성분, 수치 정보는 원본 사진, URL, 사용자 입력, OCR 결과처럼 출처가 있는 자료를 우선 사용한다.

AI가 생성한 이미지는 아래 용도에 한정한다.

```text
- 배경 분위기
- 사용 장면의 공간
- 추상적인 장식 요소
- 상품 정확도와 직접 관련 없는 보조 이미지
```

### 5.3 섹션마다 이미지 전략을 다르게 선택한다

각 섹션은 아래 네 가지 시각 전략 중 하나를 가진다.

```text
original_asset
원본 사진 또는 URL에서 추출한 이미지를 그대로 사용

cutout_composite
원본 상품 누끼를 AI 배경 또는 디자인 배경 위에 합성

generated_scene
상품 정체성 왜곡 위험이 낮은 텍스트 없는 장면 이미지를 생성

html_graphic
이미지 없이 HTML 카드, 표, 아이콘, 배경, 텍스트로 구성
```

## 6. 생성 파이프라인

최종 파이프라인은 다음 순서로 간다.

```text
1. 입력 수집
2. 자료 추출
3. 사실 카드 생성
4. 사용자 확인
5. 상세페이지 스토리보드 생성
6. 섹션별 장면 계획
7. 이미지 후보 생성/선택
8. React 상세페이지 문서 조립
9. 결과 화면 편집
10. Playwright PNG/JPG export
```

## 7. 데이터 모델 개념

상세페이지는 긴 이미지가 아니라 아래와 같은 문서 데이터로 저장한다.

```json
{
  "template": "premium_black_supplement",
  "input_quality": "high",
  "facts": [
    {
      "id": "fact-1",
      "text": "600mg x 2정",
      "source": "user_text",
      "verification_status": "confirmed"
    }
  ],
  "sections": [
    {
      "id": "hero",
      "type": "hero_dark",
      "visual_strategy": "cutout_composite",
      "image_asset_id": "asset-product-cutout",
      "generated_background_asset_id": "asset-hero-bg",
      "text": {
        "headline": "처음 만나는 쉬운 콘드로이친",
        "subcopy": "하루 한 번, 2알이면 충분합니다."
      }
    }
  ]
}
```

## 8. 사용자 확인 화면

생성 전 확인 화면에는 AI가 읽은 정보를 필드별로 보여준다.

```text
상품명
핵심 특징
스펙/성분
가격/배송
사용 가능한 이미지
확인 필요한 정보
사용하지 않을 정보
원하는 분위기
판매 채널
```

각 항목에는 신뢰도 라벨을 붙인다.

```text
확실함: 사용자가 직접 입력했거나 HTML에서 명확히 추출한 정보
확인 필요: OCR 또는 이미지 추정으로 얻은 정보
사용 금지: 출처가 없거나 과장/위험 표현 가능성이 있는 정보
```

## 9. 상세페이지 품질 목표

첨부된 좋은 예시처럼 보이려면 아래 요소가 필요하다.

```text
- 섹션별 목적이 뚜렷해야 한다.
- 어두운 섹션과 밝은 섹션의 리듬이 있어야 한다.
- 원본 이미지를 단순 반복하지 않고 크롭, 누끼, 합성, 배경 변화로 변주해야 한다.
- 제품 정보, 성분, 인증, 추천 대상, FAQ, CTA가 조판되어야 한다.
- 한글 타이포그래피가 이미지 위에 안정적으로 얹혀야 한다.
- 표와 체크리스트는 HTML로 선명하게 렌더링되어야 한다.
```

## 10. MVP 범위

처음 구현은 아래까지만 잡는다.

```text
- 사진 1장 또는 여러 장 업로드
- 자유 입력 한 박스
- AI 구조화 결과 확인 화면
- 기존 URL/텍스트 수집 파이프라인 연결
- 사실 카드 확인
- 섹션별 visual_strategy 생성
- 이미지 프롬프트의 no-text 정책 적용
- React 텍스트 레이어 상세페이지 조립
- WYSIWYG export와 연결
```

다음 단계로 미룰 항목:

```text
- 완전 자동 쇼핑몰 등록
- 고급 누끼 보정 UI
- 모든 URL의 완전 자동 수집
- 인증서/공장 자료 자동 검증
- 3D 또는 정확한 측면/후면 재구성
```

## 11. 최종 결정

Sellform의 최종 방향은 아래로 확정한다.

```text
사진 1장으로 빠르게 시작하고,
자료를 더 넣을수록 더 진짜 같은 상세페이지를 만들며,
한글은 깨지지 않는 편집 가능한 텍스트 레이어로 렌더링하고,
이미지 생성은 부족한 장면 보충용으로만 사용하는
자료 기반 조립형 AI 상세페이지 엔진.
```
