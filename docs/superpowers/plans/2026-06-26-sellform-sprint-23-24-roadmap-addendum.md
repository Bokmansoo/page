# Sellform Sprint 23~24 Roadmap Addendum

이 문서는 기존 `2026-06-23-sellform-sprint-roadmap.md`의 후속 추가 로드맵이다. 기존 로드맵 파일 일부에 인코딩 깨짐이 있어, Sprint 23~24는 별도 addendum으로 관리한다.

## Sprint 23 - LLM 웹 브라우징 기반 사실 카드 자동 생성

**목표:** 상품 링크만 입력해도 LLM 웹 검색/브라우징 보조 수집을 통해 상품 정보를 확보하고, 근거가 있는 사실 카드 후보를 생성한다.

### 범위

- 기존 `httpx` URL 직접 수집 실패 시 OpenAI Responses API web search fallback을 시도한다.
- 기본 모델은 `gpt-5.4-nano`를 사용한다.
- OpenAI 실패 시 Google/Gemini 및 deterministic fallback을 유지한다.
- web search 성공 결과를 사실 카드 후보 생성 입력으로 연결한다.
- web search 실패, API 키 없음, 빈 결과를 사용자에게 명확히 안내한다.
- AI 후보는 자동 확정하지 않고 사용자 검수를 요구한다.

### 실행계획

- `docs/superpowers/plans/2026-06-26-sellform-sprint-23-llm-web-browsing-fact-extraction-실행계획.md`

### 완료 기준

- 쿠팡/스마트스토어처럼 URL 직접 수집이 차단되어도 web search fallback을 시도한다.
- fallback 성공 시 사실 카드 후보가 생성된다.
- 실패 시 수동 입력 fallback이 명확히 표시된다.
- 백엔드 테스트와 프론트 빌드가 통과한다.

## Sprint 24 - 브라우저 보조 상품 정보 수집 UX

**목표:** 자동 URL 수집과 LLM web search로도 부족한 경우, 사용자가 실제 브라우저에서 접근 가능한 상품 정보를 편하게 복사/붙여넣고 사실 카드로 변환할 수 있게 한다.

### 범위

- URL 수집 실패 시 “상품 페이지 새 탭에서 열기” CTA를 제공한다.
- 복사한 상세 설명/스펙을 한 번에 붙여넣는 브라우저 보조 수집 패널을 제공한다.
- 붙여넣은 텍스트를 여러 사실 카드 후보로 변환한다.
- 후보는 자동 확정하지 않고 사용자 검수를 요구한다.
- 로그인/캡차/접근 제한 우회는 하지 않는다는 정책을 명확히 표시한다.

### 실행계획

- `docs/superpowers/plans/2026-06-26-sellform-sprint-24-browser-assisted-url-collection-실행계획.md`

### 완료 기준

- 사용자는 URL 수집 실패 후 다음 행동을 명확히 알 수 있다.
- 상품 페이지를 새 탭으로 열고 상세정보를 복사해 Sellform에 붙여넣을 수 있다.
- 붙여넣은 정보가 여러 사실 카드 후보로 변환된다.
- 확인된 사실 3개 이상이면 다음 단계로 진행할 수 있다.
- 프론트 빌드와 백엔드 테스트가 통과한다.

## Sprint 25 - 소셜/영상 소스 접수

**목표:** 인스타그램 릴스, 쇼츠, 틱톡, 쿠팡 숏클립처럼 영상 중심 상품 소스를 Sellform 프로젝트의 근거 자료로 안전하게 접수한다.

### 범위

- 상품 프로젝트에 영상/소셜 링크를 저장한다.
- URL host/path 기반으로 소스 유형을 `instagram`, `youtube_shorts`, `tiktok`, `coupang_shortclip`, `smartstore_clip`, `other`로 분류한다.
- 사용자가 직접 확인한 캡션, 설명, 댓글 요약, 장면 메모를 근거 자료로 남길 수 있게 한다.
- 외부 영상 다운로드, 로그인 세션 우회, 캡차 우회는 하지 않는다.

### 실행계획

- `docs/superpowers/plans/2026-06-26-sellform-sprint-25-social-video-source-intake-실행계획.md`

### 완료 기준

- 소셜/영상 URL을 프로젝트에 저장할 수 있다.
- 소스 유형이 자동 분류된다.
- 사용자는 영상 링크를 새 탭에서 열고, 확인한 설명/메모를 Sellform에 남길 수 있다.
- 백엔드 테스트와 프론트 빌드가 통과한다.

## Sprint 26 - 이미지/영상 근거 추출

**목표:** 사용자가 업로드한 상품 이미지, 스크린샷, 짧은 클립에서 OCR/멀티모달 후보를 추출해 사실 카드 후보로 만든다.

### 범위

- 업로드된 이미지/영상 자산을 근거 분석 대상으로 표시한다.
- 자산별 후보에는 `asset_id`, 근거 종류, 신뢰도, 추출 메모를 남긴다.
- 후보는 자동 확정하지 않고 사용자 검수 후 저장한다.
- 외부 플랫폼 영상 무단 다운로드와 장시간 영상 전체 분석은 하지 않는다.

### 실행계획

- `docs/superpowers/plans/2026-06-26-sellform-sprint-26-video-image-evidence-extraction-실행계획.md`

### 완료 기준

- 이미지/영상 자산에서 후보 생성 버튼을 사용할 수 있다.
- 후보에는 근거 자산과 신뢰도가 표시된다.
- 선택한 후보만 사실 카드로 저장된다.
- 백엔드 테스트와 프론트 빌드가 통과한다.

## Sprint 27 - 소셜 영상 기반 상세페이지 생성 흐름

**목표:** 소셜 영상/이미지/수동 근거에서 확인된 사실 카드를 바탕으로 생활/리빙 상품의 7단 설득형 상세페이지 초안을 생성하고 저장/export 흐름까지 연결한다.

### 범위

- 소셜 소스가 있는 프로젝트에 `영상 기반 상세페이지 생성` 모드를 제공한다.
- 영상형 7단 구조를 적용한다.
- 영상 장면 메모는 후킹과 상황 설명에만 사용한다.
- 수치, 효능, 성능 주장은 확인된 사실 카드가 있을 때만 사용한다.
- 생성 결과는 기존 page-editor, version, export 흐름으로 연결한다.

### 실행계획

- `docs/superpowers/plans/2026-06-26-sellform-sprint-27-social-video-to-detail-page-flow-실행계획.md`

### 완료 기준

- 영상 기반 상세페이지 생성 모드를 선택할 수 있다.
- 영상형 상세페이지는 7단 구조를 유지한다.
- 근거 없는 위험 주장은 생성하지 않는다.
- 생성 결과가 저장/export 흐름으로 이어진다.
- 백엔드 테스트와 프론트 빌드가 통과한다.
