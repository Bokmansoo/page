# Sprint 28 - AI 배경 비주얼 생성 기획 보강

> 이 문서는 `2026-06-26-sellform-sprint-28-ai-background-visual-generation-실행계획.md`의 보강본이다. 기존 문서에 인코딩 깨짐이 있어, 실제 구현 기준은 이 문서를 우선한다.

## 목표

현재 Sellform의 export 결과물은 한글 문구와 7단 구성은 보이지만, 흰 배경 중심이라 판매용 상세페이지 느낌이 약하다. Sprint 28의 목표는 상품명, 카테고리, 확인된 사실 카드, 선택된 판매 전략을 바탕으로 상세페이지에 어울리는 배경 비주얼 후보를 제공하고, 사용자가 선택한 배경을 미리보기와 export에 반영하는 것이다.

## 핵심 방향

1차 구현은 실제 이미지 생성 API보다 “배경 후보 시스템”을 먼저 만든다.

```text
상품명 + 카테고리 + 확인된 사실 카드 + 판매 전략
→ 배경 콘셉트 후보 2~3개 생성
→ 사용자가 하나 선택
→ page-editor 미리보기 반영
→ 긴 세로 export 이미지에 반영
```

처음부터 AI 이미지 생성 API를 붙이지 않는 이유는 다음과 같다.

- 비용과 속도 리스크가 있다.
- AI가 실제 제품과 다른 가짜 제품 이미지를 만들 수 있다.
- 타사 로고, 인증마크, 브랜드 실물 이미지가 잘못 생성될 수 있다.
- 지금 가장 큰 문제는 “배경이 전혀 없어 상세페이지처럼 보이지 않는 것”이므로, 먼저 안전한 fallback 배경으로 품질을 끌어올리는 편이 빠르다.

## 1차 구현 범위

### 1. 배경 후보 생성

생활/리빙 상품 기준으로 다음과 같은 후보를 만든다.

- 쿨링 블루 그라데이션형
- 미니멀 화이트 제품 강조형
- 여름/실내 라이프스타일 무드형

각 후보는 다음 정보를 가진다.

```json
{
  "id": "cooling-blue",
  "title": "쿨링 블루 그라데이션형",
  "description": "시원한 공기감과 휴대용 선풍기 사용 상황을 연상시키는 배경입니다.",
  "palette": ["#EAF4FF", "#DDEBFF", "#FFFFFF"],
  "style_key": "cooling_gradient",
  "safety_note": "실제 제품 이미지, 로고, 인증마크는 생성하지 않습니다."
}
```

### 2. page-editor UI

page-editor 우측 디자인 패널에 다음 UI를 추가한다.

- `AI 배경 후보 만들기` 버튼
- 배경 후보 카드 2~3개
- 각 카드의 제목, 설명, 색상 팔레트, 추천 이유
- `선택` 버튼
- 선택된 후보 표시

선택된 배경은 중앙 모바일 미리보기에 즉시 반영한다.

### 3. export 반영

선택한 배경은 긴 세로 이미지 export에 반영한다.

기본 방향:

- 상단 히어로 섹션에는 선택한 배경의 색상/그라데이션을 강하게 반영한다.
- 본문 섹션은 완전 흰색만 반복하지 않고, 아주 연한 배경색과 카드형 레이아웃을 섞는다.
- 상품 정보/인증 정보 영역은 가독성을 위해 흰색 카드에 가깝게 유지한다.

## 2차 구현 범위

실제 AI 이미지 생성은 2차로 미룬다.

추후 고도화 시 다음을 붙인다.

- OpenAI 이미지 생성 또는 다른 이미지 Provider 연결
- 상품 이미지가 업로드된 경우 상품 사진 기반 배경 합성
- 상품 이미지가 없는 경우 추상/라이프스타일 배경만 생성
- 생성 이미지 후보 저장 및 재사용
- 이미지 생성 비용/실패/검수 로그 기록

## 안전 규칙

AI 배경 생성 또는 후보 추천은 다음 규칙을 지킨다.

- 타사 로고를 생성하지 않는다.
- 인증마크, KC 마크, 쿠팡/네이버 로고를 생성하지 않는다.
- 실제 제품 사진이 없을 때 제품 실물을 임의로 만들지 않는다.
- 상품 사실과 충돌하는 장면을 만들지 않는다.
- 텍스트가 들어간 이미지를 만들지 않는다. 문구는 Sellform의 섹션 렌더링에서 처리한다.

## 구현 단계

### Sprint 28.1 - 배경 후보 서비스

- `backend/src/services/visual_background_service.py` 생성
- 상품명, 카테고리, 전략, 사실 카드 기반 후보 생성
- 이미지 API가 없어도 fallback 후보 반환
- 생활/리빙 기본 후보 3개 제공

### Sprint 28.2 - API 연결

- `POST /api/v1/projects/{project_id}/visual-backgrounds/generate`
- `POST /api/v1/projects/{project_id}/visual-backgrounds/{candidate_id}/select`
- 선택된 배경 후보를 프로젝트 또는 페이지 버전에 저장

### Sprint 28.3 - page-editor UI

- 우측 디자인 패널에 배경 후보 생성/선택 UI 추가
- 선택된 후보를 모바일 미리보기에 반영
- 선택되지 않은 경우 기본 흰 배경 유지

### Sprint 28.4 - export 반영

- `export_service.py`가 선택된 배경 팔레트를 읽는다.
- 긴 세로 이미지 상단/섹션 배경에 팔레트를 반영한다.
- 한글 가독성을 해치지 않도록 배경 대비를 낮게 유지한다.

### Sprint 28.5 - 문서와 검증

- 테스트 로그 작성
- 코드리뷰 문서 작성
- 트러블슈팅 문서 작성
- 실제 루메나 상품으로 export 전후 비교

## 테스트 기준

- 배경 후보가 2개 이상 생성된다.
- 후보에는 title, description, palette, style_key가 있다.
- page-editor에서 후보 선택 시 미리보기가 바뀐다.
- export 결과물에 선택한 배경이 반영된다.
- 배경이 있어도 한글 본문 가독성이 유지된다.
- 이미지 생성 API가 없어도 fallback으로 동작한다.
- 백엔드 테스트와 프론트 빌드가 통과한다.

## 완료 기준

- 루메나 휴대용 무선 냉각선풍기 프로젝트에서 배경 후보를 생성할 수 있다.
- 사용자는 후보 중 하나를 선택할 수 있다.
- 선택한 배경이 상세페이지 미리보기와 export 결과물에 반영된다.
- 현재 흰 배경 문서형 결과물보다 판매용 상세페이지에 가까워진다.
- 실제 이미지 생성 API 없이도 안정적으로 동작한다.
