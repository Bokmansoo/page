# 트러블슈팅: Sellform Sprint 9

## 1. 개요
Sprint 9의 1.0 실사용 검증 과정 중, 12개 실상품 데이터를 대상으로 백엔드 API와 프론트엔드 E2E 흐름을 검토했습니다. 

## 2. 결함 검토 및 분석
시드 데이터 E2E 실행 결과, 서버 다운이나 UI 렌더링 중단과 같은 **시스템적인 코드 결함(Blocker/Major)**은 단 한 건도 발견되지 않았습니다.

검출된 Blocker/Major 경고들은 모두 기획서 스펙(전성분 고시 누락, 아동용품 KC인증번호 누락, 의학적/치료 목적 허위 과장 카피, 질병 효능 표방 등)에 근거하여 **Compliance Checker가 요구조건대로 정확하게 차단/경고를 해주는 정상적인 동작**입니다.

## 3. 기록된 비즈니스 위반 검수 항목 (정상 필터링 사례)

### B1. 아동용품 KC 안전인증 누락 (Blocker)
- **상품 ID**: S9-LIVING-02 (친환경 무독성 아동용 식기 세트)
- **증상**: 이미지 export 차단됨.
- **원인**: 아동/어린이용 제품으로 분류되었으나, 입력 텍스트 또는 사실 카드에 필수 KC 안전인증 정보(인증번호 등)가 없어 Blocker 등급으로 자동 판정.
- **해결 방안**: 사실 카드에 KC 안전인증 마크와 인증 번호를 수동 추가한 후 재검수하여 해결 가능.

### B2. 화장품 의학적/치료 목적 과장 표현 (Blocker)
- **상품 ID**: S9-BEAUTY-02 (비타민 C 브라이트닝 크림)
- **증상**: 이미지 export 차단됨.
- **원인**: 화장품 법상 금지된 "아토피 완벽 치료", "여드름균 100% 사멸" 등의 단정적/의학적 효능 문구가 페이지 카피에 그대로 포함됨.
- **해결 방안**: 상세페이지 편집기에서 해당 문구를 미용 효능("피부 생기 부여", "피부 진정 케어" 등)으로 순화하여 부분 재생성 후 재검수.

### B3. 건강식품 질병 치료/예방 관련 과대광고 (Blocker)
- **상품 ID**: S9-FOOD-02 (기력 충전 보양 장어즙)
- **증상**: 이미지 export 차단됨.
- **원인**: 일반 건강식품임에도 불구하고 "고혈압, 당뇨 예방 및 완벽 치료 특효약" 등 질병에 대한 치료 효능 표현이 포함되어 Blocker 판정.
- **해결 방안**: 질병 치료 효능에 대한 문구를 삭제하거나 건강 증진 수준으로 문구를 수정하여 재검수.

## 4. 운영·입력 흐름에서 발견된 보류 항목

### M1. AI 분석 Timeout으로 인한 draft 잔류

- **상품 ID**: S9-FASHION-03 (빈티지 워싱 데님 자켓)
- **증상**: 외부 AI 분석 Timeout으로 상품이 `draft` 상태에 남고 이미지 export 및 공개 발행까지 진행되지 못함.
- **원인**: AI API 호출 실패 또는 응답 지연 상황에서 자동 재시도/대체 입력 흐름이 충분히 강하지 않음.
- **해결 방안**:
  - 단기: 수동 입력과 재시도 버튼으로 복구한다.
  - Sprint 10 이관: AI 사실 카드 자동 추출 단계에서 timeout, 빈 결과, 스키마 오류를 명확히 분리하고 deterministic fallback 또는 부분 후보 생성 흐름을 강화한다.

### M2. 입력 부족으로 인한 checking/draft 잔류

- **상품 ID**: S9-BEAUTY-03, S9-FOOD-03 등
- **증상**: 전성분, 필수 고시 정보, 원재료, 이미지 근거가 부족한 상품이 export까지 진행되지 못함.
- **원인**: 사용자가 상품 사실을 수기로 채워야 하는 부담이 크고, 링크·이미지에서 자동으로 채워지는 사실 후보가 충분하지 않음.
- **해결 방안**:
  - 단기: 사실 카드에 필수 고시 정보와 근거 텍스트를 수동 보완한다.
  - Sprint 10 이관: 공급처 URL, 복사 텍스트, 업로드 이미지에서 사실 후보를 자동 생성하고 사용자는 검수·확정 중심으로 작업하도록 개선한다.

### M3. 원래 export 10개 기준 미충족

- **증상**: Sprint 9 초기 완료 기준은 `10개 이상 이미지 export 생성`이었으나 실제 성공 수는 5개였다.
- **원인**: 일부 상품이 compliance 차단, 입력 부족, AI Timeout으로 보류되었다. 이 중 compliance 차단은 시스템 결함이 아니라 위험 상품을 막는 정상 동작이다.
- **판정**: Sprint 9는 “완전 승인”이 아니라 “조건부 승인”으로 마감한다.
- **후속 조치**: Sprint 10에서 입력 자동화와 사실 후보 추출 품질을 높인 뒤, 보류 상품을 재검증한다.

## 5. Sprint 10 이관 항목

- 공급처 URL, 복사 텍스트, 업로드 이미지에서 사실 카드 후보를 자동 생성한다.
- 자동 생성 후보는 기본 `unknown` 또는 `needs_revision` 상태로 저장하고, 사용자가 `confirmed`로 승인한 사실만 상세페이지 생성에 사용한다.
- URL 수집 실패, AI Timeout, 빈 결과, 스키마 오류를 사용자에게 구분해서 안내한다.
- compliance 차단 사례는 실패가 아니라 정상 필터링으로 표시하고, 수정해야 할 사실/문구를 더 구체적으로 안내한다.

## 6. 결론

시스템적인 코드 오류나 리소스 누수는 발견되지 않았으므로 즉각적인 핫픽스는 불필요하다. 다만 원래 Sprint 9의 `10개 이상 export` 기준은 충족하지 못했으므로 Sprint 9는 조건부 완료로 마감한다. 미충족 항목은 입력 자동화와 사실 카드 자동 추출 품질 문제로 분류하고 Sprint 10에서 보완한다.
