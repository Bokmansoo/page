# Sprint 31 - 커머스 컷 렌더링 트러블슈팅

본 문서는 Sprint 31 구현 과정에서 발생한 핵심 문제 상황과 해결 방안에 대한 기록입니다.

---

## 1. Pillow 컷 이미지 크롭 시 찌그러짐 현상
- **현상**: 상품 컷 이미지가 컷 크기에 맞춰 늘어나거나 줄어들 때 종횡비가 파괴되어 뚱뚱하거나 길쭉하게 보이는 현상이 발견되었습니다.
- **원인**: Pillow의 `resize` 함수를 가로/세로 고정 비율로 단순 적용하여 드로잉했기 때문입니다.
- **해결책**: Pillow의 `ImageOps.fit` 유틸리티를 적용하여 원본의 종횡비를 보존하며 중심점을 기준으로 지정한 영역(764x420)에 가득 차게 크롭하도록 수정했습니다.

## 2. 드로잉 마스킹 둥근 사각형 찌꺼기 현상
- **현상**: 둥글게 마스킹한 이미지를 paste할 때 테두리 부근에 검은색이나 흰색의 깨진 픽셀(안티앨리어싱 잔재)이 발생하는 현상이 있었습니다.
- **원인**: 마스크 이미지 모드가 1-bit(모노크롬)로 설정되어 경계면 보정이 안 되었기 때문입니다.
- **해결책**: 마스크 이미지를 8-bit 그레이스케일(`"L"`) 모드로 생성하고, `ImageDraw.Draw(mask).rounded_rectangle` 을 통해 부드럽게 둥근 안티앨리어싱 마스크가 반영되도록 처리했습니다.

## 3. 프론트엔드 strict 린트 에러 (Next.js 빌드 차단)
- **현상**: page.tsx 수정 후 Next.js 빌드 시 `isBgActive is assigned a value but never used` 및 `let bgStyle ... prefer-const` 오류로 빌드가 취소되었습니다.
- **원인**: TypeScript 및 ESLint 룰이 엄격하게 잡혀 있어 미사용 변수 선언이 컴파일을 중단시켰습니다.
- **해결책**: 에디터 컴포넌트 내에서 미사용 변수를 즉시 지워주고, 재할당되지 않는 변수를 `const` 로 정정 선언하여 빌드 린트를 패스시켰습니다.
