# 운영·복구 런북

사람이 바뀌어도 같은 방식으로 대응해야 하는 운영 절차를 기록한다. 예: AI 작업 대기열 정체, 렌더링 실패, 파일 저장소 장애, 배포 롤백.

각 런북에는 발동 조건, 영향 확인 방법, 즉시 완화 절차, 복구 절차, 검증, 에스컬레이션 기준, 사후 기록 위치를 포함한다.
